1. Документация RusNAS

Документация RusNAS

Добро пожаловать в руководство пользователя RusNAS -- российской платформы для сетевых хранилищ данных корпоративного класса.

RusNAS предоставляет полнофункциональную замену зарубежным NAS-решениям (Synology DSM, QNAP QTS, TrueNAS) с веб-интерфейсом управления, поддержкой всех основных сетевых протоколов и продвинутыми средствами защиты данных.

Возможности платформы

Категория Что входит
Хранилище SMB/NFS шары, iSCSI, WORM, FTP, WebDAV, файловый менеджер
RAID Уровни 0, 1, 5, 6, 10, JBOD с горячей заменой дисков
Файловая система Btrfs со снапшотами, дедупликацией и онлайн-расширением
Защита Guard (антишифровальщик), снапшоты, репликация, WORM
Мониторинг Дашборд, ночной отчёт, анализатор пространства, S.M.A.R.T.
Сеть Bonding, VLAN, сертификаты Let's Encrypt, DDNS, диагностика
ИБП NUT (Network UPS Tools) с авто-отключением
Оптимизация Авто-тюнинг 12 параметров, SSD-кеширование, резервный режим
Приложения Каталог из 10 контейнерных приложений (Nextcloud, Immich и др.)
AI-ассистент Интеграция с Yandex GPT и Claude для управления через чат

2. Первый вход в систему

Первый вход в систему

После установки RusNAS веб-интерфейс управления доступен через браузер. Эта страница описывает, как войти в систему и выполнить первоначальную настройку.

Требования

  • Компьютер или мобильное устройство в одной сети с сервером RusNAS
  • Современный браузер: Chrome, Firefox, Safari или Edge
  • IP-адрес сервера RusNAS (по умолчанию назначается через DHCP)

Открытие веб-интерфейса

  1. Откройте браузер и введите в адресной строке:

    https://<IP-адрес>:9090

    Например: https://192.168.1.100:9090

  2. Браузер может показать предупреждение о сертификате безопасности. Это нормально для первого подключения -- нажмите "Дополнительно" и "Перейти на сайт".

!!! note "Порт 9090" Веб-интерфейс RusNAS работает на порту 9090. Не забудьте указать его после IP-адреса через двоеточие.

Форма входа

На странице входа вы увидите два поля:

Поле Описание
Имя пользователя Системное имя пользователя Linux. По умолчанию: rusnas
Пароль Пароль учётной записи, заданный при установке

Введите учётные данные и нажмите кнопку "Войти".

!!! warning "Внимание" Учётные данные для входа -- это системная учётная запись Linux, а не отдельный аккаунт веб-интерфейса. Пароль задаётся при установке ОС.

Что вы увидите после входа

После успешной авторизации откроется Дашборд -- главная страница мониторинга. На ней отображаются:

  • Текущая загрузка CPU и оперативной памяти
  • Состояние дисков и RAID-массивов
  • Статус Guard (антишифровальщик)
  • Состояние ИБП (если подключён)
  • Ночной отчёт (сводка за последние 8 часов)

Первоначальная настройка

После первого входа рекомендуется выполнить следующие шаги в указанном порядке:

  1. Включить административный доступ -- необходимо для большинства операций. См. Административный доступ.

  2. Проверить диски -- откройте страницу Диски и RAID и убедитесь, что все физические диски обнаружены.

  3. Создать RAID-массив -- объедините диски в массив нужного уровня. См. Создание массива.

  4. Создать общие папки -- настройте SMB и/или NFS шары для доступа к данным по сети. См. Общие папки.

  5. Создать пользователей -- добавьте учётные записи для сотрудников. См. Пользователи и группы.

  6. Настроить Guard -- включите защиту от шифровальщиков. См. Guard: обзор.

  7. Настроить снапшоты -- включите автоматическое создание снимков данных. См. Расписание снапшотов.

!!! tip "Совет" Все настройки можно менять в любое время. Последовательность выше -- рекомендация для быстрого запуска.

Смена языка интерфейса

Интерфейс Cockpit автоматически определяет язык из настроек браузера. Если интерфейс отображается на английском:

  1. Откройте настройки браузера
  2. Добавьте русский язык в список предпочтительных
  3. Переместите его на первое место
  4. Обновите страницу RusNAS

Доступ с мобильных устройств

Веб-интерфейс RusNAS полностью адаптирован для мобильных устройств. Откройте тот же адрес https://<IP>:9090 в мобильном браузере. Навигация и основные операции доступны с любого экрана.

3. Обзор интерфейса

Обзор интерфейса

Веб-интерфейс RusNAS построен на платформе Cockpit и состоит из боковой панели навигации, верхней панели и рабочей области. Все страницы выполнены в единой тёмной теме.

Боковая панель навигации

Слева расположена вертикальная панель со ссылками на все разделы системы. Активная страница подсвечена акцентным цветом (оранжевый). Панель содержит 12 страниц:

Значок Страница Назначение
Дашборд Дашборд Мониторинг системы, ночной отчёт
Хранилище Хранилище SMB/NFS шары, iSCSI, WORM, FTP, WebDAV
Диски Диски и RAID Управление массивами, S.M.A.R.T., SSD-кеш
Пользователи Пользователи Учётные записи и группы
Guard Guard Антишифровальщик
Снапшоты Снапшоты Btrfs-снимки и репликация
Дедупликация Дедупликация Экономия пространства (duperemove)
ИБП ИБП Управление источником бесперебойного питания
Анализ Анализ Анализатор дискового пространства
Сеть Сеть Сетевые интерфейсы, сертификаты, домен
AI AI Ассистент Чат с AI для управления системой
Оптимизация Оптимизация Авто-тюнинг производительности

Для перехода между страницами нажмите на соответствующий пункт в боковой панели.

Верхняя панель

В верхней части каждой страницы отображается информационная строка:

  • IP-адрес сервера
  • Время работы (uptime) с момента последней перезагрузки
  • Версия RusNAS
  • Текущая дата и время

Справа расположен индикатор административного доступа (замок) и кнопка выхода из системы.

Элементы интерфейса

Карточки

Основной элемент отображения информации. Карточки содержат:

  • Заголовок с иконкой
  • Статус (цветной индикатор)
  • Ключевые метрики
  • Кнопки действий

На дашборде карточки расположены в сетке и обновляются автоматически каждые несколько секунд.

Таблицы

Списки данных (шары, пользователи, снапшоты, события) отображаются в таблицах с возможностью:

  • Сортировки по столбцам
  • Фильтрации по параметрам
  • Постраничной навигации (для больших списков)

На мобильных устройствах таблицы прокручиваются горизонтально.

Вкладки

Многие страницы разделены на вкладки для логической группировки функций. Например:

  • Хранилище: Шары | iSCSI | WORM | Сервисы
  • Guard: Обзор | Журнал событий | Настройки
  • Снапшоты: Снапшоты | Расписание | Репликация

Нажмите на название вкладки для переключения. Активная вкладка подчёркнута.

Модальные окна

Формы создания и редактирования открываются в модальных окнах поверх основной страницы. Модальное окно содержит:

  • Заголовок операции
  • Поля формы с подписями
  • Кнопки "Сохранить" и "Отмена"

Для закрытия окна нажмите "Отмена", крестик в углу или клавишу Escape.

Индикаторы статуса

Цветные точки и бейджи используются для отображения состояния:

Цвет Значение
Зелёный Работает нормально, активен
Жёлтый Предупреждение, требует внимания
Красный Ошибка, критическое состояние
Серый Выключен, не подключён
Синий В процессе (зарядка, синхронизация)

Тёмная тема

Интерфейс RusNAS использует тёмную цветовую схему. Это снижает нагрузку на глаза при длительной работе и является стандартом для серверных платформ. Основные цвета:

  • Фон: тёмно-серый
  • Текст: светло-серый и белый
  • Акценты: оранжевый (брендовый цвет RusNAS)
  • Ошибки: красный
  • Успех: зелёный

Обновление данных

Большинство страниц обновляют данные автоматически:

  • Дашборд: каждые 2 секунды (CPU, RAM, сеть), каждые 10 секунд (Guard), каждые 30 секунд (ИБП)
  • Диски: каждые 4 секунды при активной перестройке массива
  • Guard: каждые 5 секунд (журнал событий)

Для принудительного обновления любой страницы нажмите F5 или кнопку обновления в браузере.

!!! tip "Совет" Если страница выглядит некорректно или данные не обновляются, попробуйте очистить кеш браузера (Ctrl+Shift+R) и обновить страницу.

4. Административный доступ

Административный доступ

Для выполнения большинства операций по настройке RusNAS необходимо включить административный (привилегированный) доступ. Без него вы сможете только просматривать информацию, но не изменять конфигурацию.

Зачем нужен административный доступ

RusNAS работает с системными сервисами Linux: RAID-массивами, файловыми системами, сетевыми протоколами. Для их настройки требуются права суперпользователя (root). Административный доступ предоставляет эти права через безопасный механизм Cockpit.

Как включить

  1. Войдите в веб-интерфейс RusNAS
  2. В верхней правой части экрана найдите значок замка или надпись "Ограниченный доступ"
  3. Нажмите на него
  4. В появившемся диалоге нажмите "Включить административный доступ"
  5. Введите пароль вашей учётной записи
  6. Нажмите "Подтвердить"

После включения значок замка изменится, и все операции управления станут доступны.

!!! warning "Внимание" Административный доступ действует до конца текущей сессии. При выходе из системы или закрытии браузера его нужно будет включить снова.

Просмотр дашборда и метрик Нет
Создание/удаление RAID-массива Да
Монтирование/размонтирование Да
Создание/редактирование шар Да
Управление iSCSI Да
Настройки Guard Да
Создание/удаление снапшотов Да
Управление пользователями Да
Сетевые настройки Да
Установка приложений Да
Настройка ИБП Да
Оптимизация производительности Да
Просмотр журналов Нет
Просмотр статуса Guard Нет

Безопасность

Административный доступ в RusNAS реализован через стандартный механизм Cockpit (polkit), который:

  • Запрашивает пароль при каждом включении
  • Автоматически отключается при завершении сессии
  • Не сохраняет привилегии между сессиями
  • Записывает все действия в системный журнал

!!! danger "Важно" Не оставляйте веб-интерфейс с включённым административным доступом без присмотра. Любой, кто имеет доступ к открытой сессии браузера, сможет изменять конфигурацию сервера.

Если административный доступ не включается

Возможные причины:

  1. Неверный пароль -- убедитесь, что вводите пароль системной учётной записи, а не пароль Guard или другой пароль
  2. Учётная запись не в группе sudo -- пользователь должен состоять в группе sudo для получения административных прав. Обратитесь к администратору сервера
  3. Проблема с polkit -- перезагрузите страницу и попробуйте снова

!!! tip "Совет" На странице Диски и RAID система автоматически напомнит о необходимости включить административный доступ, если он не активен.

5. Общие папки (SMB/NFS)

Общие папки (SMB/NFS)

Общие папки -- основной способ предоставления доступа к данным на RusNAS по сети. Поддерживаются два протокола: SMB (для Windows и macOS) и NFS (для Linux-систем).

Где найти

Откройте страницу Хранилище в боковой панели. Первая вкладка "Шары" отображает все настроенные общие папки.

Статус сервисов

В верхней части вкладки отображаются индикаторы статуса:

  • SMB -- зелёная точка означает, что служба Samba запущена
  • NFS -- зелёная точка означает, что NFS-сервер запущен

Если точка серая, соответствующий сервис не запущен и шары по этому протоколу недоступны.

Таблица шар

Все настроенные шары отображаются в единой таблице:

Столбец Описание
Имя Название шары (отображается клиентам при подключении)
Путь Директория на сервере, к которой предоставлен доступ
Протоколы Значки SMB и/или NFS, показывающие, по каким протоколам доступна шара
Действия Кнопки редактирования и удаления

Создание общей папки

  1. Нажмите кнопку "+ Создать шару" над таблицей
  2. Откроется модальное окно с тремя вкладками: Основные, SMB, NFS

Вкладка "Основные"

Поле Описание
Имя шары Уникальное имя, которое увидят пользователи при подключении. Латиница, без пробелов
Том Выпадающий список смонтированных RAID-массивов. Шара будет создана на выбранном томе
Путь Подпапка внутри тома (необязательно). Если не указана, создаётся папка с именем шары
Описание Комментарий для удобства администратора

Вкладка "SMB"

Включите переключатель "Включить SMB" для предоставления доступа по протоколу SMB.

Параметр Описание
Доступ для гостей Разрешить подключение без логина/пароля. По умолчанию выключен
Только чтение Запретить запись в шару. По умолчанию выключен
Скрытая шара Шара не отображается при обзоре сети, но доступна по прямому пути
Допустимые пользователи Список пользователей, которым разрешён доступ. Пусто = доступ для всех

Вкладка "NFS"

Включите переключатель "Включить NFS" для предоставления доступа по протоколу NFS.

Параметр Описание
Разрешённые сети IP-адреса или подсети, которым разрешён доступ (например, 192.168.1.0/24)
Режим записи rw (чтение и запись) или ro (только чтение)
Root Squash Преобразование root-доступа клиента в непривилегированный. Рекомендуется оставить включённым
  1. Заполните необходимые поля и нажмите "Сохранить"

!!! note "Примечание" Одна шара может быть доступна одновременно по SMB и NFS. Включите оба протокола, если в сети есть и Windows, и Linux-клиенты.

Редактирование шары

  1. Найдите нужную шару в таблице
  2. Нажмите кнопку "Редактировать" (значок карандаша) в строке шары
  3. Модальное окно откроется с текущими настройками
  4. Измените нужные параметры
  5. Нажмите "Сохранить"

!!! warning "Внимание" При редактировании нельзя изменить том, на котором расположена шара. Для переноса на другой том удалите шару и создайте новую.

Удаление шары

  1. Нажмите кнопку "Удалить" (значок корзины) в строке шары
  2. Подтвердите удаление в диалоговом окне
  3. Шара будет удалена из конфигурации SMB и/или NFS

!!! danger "Важно" Удаление шары убирает только сетевой доступ к папке. Сами файлы на диске не удаляются. Чтобы очистить данные, удалите папку вручную через файловый менеджер или SSH.

Подключение с клиентских устройств

Windows (SMB)

  1. Откройте Проводник
  2. В адресной строке введите: \\<IP-адрес-RusNAS>\<имя-шары>
  3. Введите логин и пароль пользователя RusNAS

macOS (SMB)

  1. В Finder выберите "Подключение к серверу" (Cmd+K)
  2. Введите: smb://<IP-адрес-RusNAS>/<имя-шары>
  3. Введите учётные данные

Linux (NFS)

Смонтируйте шару командой:

sudo mount -t nfs <IP-адрес-RusNAS>:/<путь-шары> /mnt/точка-монтирования

!!! tip "Совет" Для автоматического подключения при загрузке добавьте запись в /etc/fstab на клиентском компьютере.

6. iSCSI-хранилище

iSCSI-хранилище

iSCSI позволяет предоставить блочное хранилище RusNAS удалённым серверам по сети. В отличие от SMB/NFS (файловый доступ), iSCSI работает на уровне блоков -- удалённый сервер видит подключённое хранилище как локальный диск.

Когда использовать iSCSI

  • Виртуализация (VMware, Hyper-V, Proxmox) -- хранилище для виртуальных машин
  • Базы данных -- блочный доступ обеспечивает лучшую производительность
  • Кластерные системы -- общее хранилище для нескольких серверов

Где найти

Откройте страницу Хранилище и перейдите на вкладку "iSCSI". Страница разделена на два раздела:

  • LUN Manager -- управление логическими номерами устройств (образы дисков)
  • Target Manager -- управление целями (точками подключения)

Основные понятия

Термин Описание
LUN Logical Unit Number -- файл-образ или блочное устройство, которое представляется клиенту как диск
Target Точка подключения с уникальным именем IQN, к которой подключаются клиенты
IQN iSCSI Qualified Name -- уникальный идентификатор формата iqn.2026-01.ru.rusnas:storage.target1
Backstore Хранилище данных для LUN: файловый образ (fileio) или блочное устройство (block)

Создание LUN

  1. В разделе LUN Manager нажмите "+ Создать LUN"
  2. Заполните параметры:
Поле Описание
Имя Уникальное имя LUN (латиница, без пробелов)
Тип Fileio -- файл-образ на файловой системе (рекомендуется). Block -- прямой доступ к блочному устройству
Путь/Устройство Для fileio: путь к файлу-образу (например, /mnt/data/iscsi/disk1.img). Для block: устройство (например, /dev/sdf)
Размер Размер LUN в гигабайтах (только для fileio)
  1. Нажмите "Создать"

!!! note "Примечание" Тип fileio подходит для большинства задач. Тип block используйте только если вам нужна максимальная производительность и вы готовы выделить отдельный диск.

Создание Target

  1. В разделе Target Manager нажмите "+ Создать Target"
  2. Заполните параметры:
Поле Описание
Имя IQN Уникальный идентификатор. Система предложит имя автоматически
Привязанные LUN Выберите один или несколько LUN для привязки к этому Target
  1. Нажмите "Создать"

Привязка LUN к Target

Если при создании Target вы не выбрали LUN, или хотите добавить дополнительные:

  1. Найдите нужный Target в списке
  2. Нажмите "Редактировать"
  3. В разделе привязанных LUN отметьте нужные
  4. Сохраните изменения

Удаление

Удаление LUN

  1. Нажмите "Удалить" рядом с LUN
  2. Если LUN привязан к Target, кнопка будет неактивна -- сначала отвяжите LUN от всех Target

!!! warning "Внимание" При удалении LUN типа fileio система предложит также удалить файл-образ с диска. По умолчанию файл сохраняется.

Удаление Target

  1. Нажмите "Удалить" рядом с Target
  2. Подтвердите удаление. Привязанные LUN не удаляются, только отвязываются.

Подключение с клиентского сервера

Windows

  1. Откройте "Инициатор iSCSI" (встроен в Windows Server и Windows Pro)
  2. На вкладке "Обнаружение" добавьте IP-адрес RusNAS
  3. На вкладке "Цели" выберите найденный Target и нажмите "Подключить"
  4. Откройте "Управление дисками" -- новый диск появится как неинициализированный
  5. Инициализируйте диск и создайте том

Linux

# Обнаружение Target
sudo iscsiadm -m discovery -t sendtargets -p <IP-RusNAS>

# Подключение
sudo iscsiadm -m node --login

# Новый диск появится как /dev/sdX
lsblk

!!! tip "Совет" Для автоматического подключения iSCSI при загрузке Linux-клиента включите службу iscsid и iscsi через systemd.

См. также: Общие папки SMB/NFS | WORM-защита

7. WORM-защита

WORM-защита

WORM (Write Once, Read Many) -- механизм защиты файлов от изменения и удаления. После помещения файла в WORM-защищённую директорию его невозможно изменить, переименовать или удалить до истечения срока хранения.

Когда использовать WORM

  • Хранение бухгалтерской и финансовой документации
  • Соответствие требованиям регуляторов к хранению данных
  • Защита архивов от случайного или преднамеренного удаления
  • Хранение юридически значимых документов

Где найти

Откройте страницу Хранилище и перейдите на вкладку "WORM".

Принцип работы

WORM в RusNAS работает на уровне директорий:

  1. Вы указываете путь к директории, которую нужно защитить
  2. Система устанавливает на файлы в этой директории атрибут неизменяемости
  3. Новые файлы можно добавлять в директорию
  4. Существующие файлы нельзя изменить или удалить
  5. Снять защиту может только администратор через интерфейс RusNAS

Добавление WORM-пути

  1. На вкладке "WORM" нажмите "+ Добавить путь"
  2. В модальном окне укажите:
Поле Описание
Путь Полный путь к директории на смонтированном томе (например, /mnt/data/archive)
  1. Нажмите "Сохранить"

!!! warning "Внимание" Перед включением WORM убедитесь, что все нужные файлы уже загружены в директорию в правильном виде. После включения защиты исправить файлы будет невозможно.

Список защищённых путей

Таблица отображает все директории с активной WORM-защитой:

Столбец Описание
Путь Защищённая директория
Статус Активна / Неактивна
Действия Удаление защиты

Удаление WORM-защиты

  1. Найдите нужный путь в таблице
  2. Нажмите кнопку "Удалить"
  3. Подтвердите действие

После удаления WORM-защиты файлы в директории снова можно изменять и удалять.

!!! danger "Важно" Удаление WORM-защиты -- привилегированная операция. Убедитесь, что снятие защиты не нарушает требования вашей организации к хранению данных.

Ограничения

  • WORM работает только на смонтированных томах Btrfs
  • Защита применяется к директории, а не к отдельным файлам
  • Новые файлы, добавленные в защищённую директорию, автоматически получают защиту
  • Поддиректории также защищаются

!!! tip "Совет" Для максимальной защиты данных комбинируйте WORM с автоматическими снапшотами и Guard.

8. Сервисы FTP и WebDAV

Сервисы FTP и WebDAV

RusNAS поддерживает дополнительные протоколы доступа к файлам: FTP для передачи файлов и WebDAV для работы через веб-браузер и совместимые клиенты.

Где найти

Откройте страницу Хранилище и перейдите на вкладку "Сервисы". На странице отображаются две карточки: FTP и WebDAV.

FTP

Статус и управление

В верхней части карточки FTP отображается:

  • Статус сервиса -- зелёный (запущен) или серый (остановлен)
  • Кнопки управления -- "Запустить" / "Остановить"

Настройка FTP

Нажмите "Настройки" на карточке FTP. Доступные параметры:

Параметр Описание
Анонимный доступ Разрешить подключение без логина. По умолчанию выключен
Запись для анонимных Разрешить анонимным пользователям загружать файлы. Работает только при включённом анонимном доступе

WebDAV

WebDAV позволяет работать с файлами через веб-протокол HTTP/HTTPS. Поддерживается монтирование как сетевого диска в Windows, macOS и Linux.

Статус и управление

В верхней части карточки WebDAV отображается:

  • Статус сервиса -- зелёный (запущен) или серый (остановлен)

Настройка WebDAV

Нажмите "Настройки" на карточке WebDAV для изменения параметров подключения.

Управление пользователями WebDAV

WebDAV использует собственную систему аутентификации (Digest), отдельную от системных учётных записей.

Добавление пользователя

  2. Нажмите "+ Добавить пользователя"
  3. Введите имя пользователя и пароль
  4. Нажмите "Создать"

  1. Найдите пользователя в списке
  2. Нажмите кнопку "Удалить" рядом с именем
  3. Подтвердите удаление

!!! note "Примечание" Пользователи WebDAV не связаны с системными пользователями RusNAS. Это отдельные учётные записи для доступа к файлам через WebDAV.

Windows

  2. В поле адреса введите: https://<IP-адрес-RusNAS>/webdav/

  3. Введите учётные данные

Linux

!!! tip "Совет" WebDAV через HTTPS -- безопасная альтернатива FTP для доступа к файлам через интернет. Для HTTPS необходим TLS-сертификат, который можно настроить на странице Сеть.

FileBrowser -- встроенный веб-файловый менеджер RusNAS. Позволяет просматривать, загружать и скачивать файлы через браузер без необходимости настройки клиентских программ.

10. Создание RAID-массива

Создание RAID-массива

RAID-массив объединяет несколько физических дисков в единое хранилище с заданным уровнем отказоустойчивости и производительности. Создание массива -- первый шаг после установки RusNAS.

Уровни RAID

Уровень Мин. дисков Отказоустойчивость Ёмкость Производительность Рекомендация
JBOD 1 Нет 100% Обычная Тестирование, временное хранение
RAID 0 2 Нет 100% Максимальная Только для некритичных данных
RAID 1 2 1 диск 50% Хорошая для чтения Системный диск, малые объёмы
RAID 5 3 1 диск (N-1)/N Хорошая Универсальный выбор
RAID 6 4 2 диска (N-2)/N Средняя Большие массивы, критичные данные
RAID 10 4 1 диск в каждой паре 50% Отличная Базы данных, виртуализация

!!! warning "Внимание" RAID 0 и JBOD не обеспечивают защиту от потери данных. При выходе из строя любого диска все данные будут потеряны. Используйте эти уровни только для данных, которые можно восстановить из другого источника.

Подготовка

Перед созданием массива убедитесь:

  1. Диски установлены -- откройте страницу "Диски и RAID" и проверьте, что все физические диски отображаются в разделе "Физические диски"
  2. Административный доступ включён -- см. Административный доступ
  3. Диски не заняты -- диски, уже входящие в массив, не могут быть использованы повторно

!!! tip "Совет" Если новый диск не отображается, нажмите кнопку "Сканировать SCSI" на странице дисков. Система выполнит поиск новых подключённых устройств.

Пошаговое создание (2-шаговый мастер)

Шаг 1: Выбор уровня и дисков

  1. На странице "Диски и RAID" нажмите кнопку "+ Создать массив"
  2. Откроется мастер создания:
Поле Описание
Уровень RAID Выберите из выпадающего списка (JBOD, RAID 0, 1, 5, 6, 10)
Доступные диски Список дисков с указанием модели, серийного номера, размера и состояния S.M.A.R.T.
  1. Отметьте галочками диски, которые войдут в массив
  2. Система покажет расчётную ёмкость массива
  3. Нажмите "Далее"

!!! danger "Важно" Создание RAID-массива уничтожит все данные на выбранных дисках. Убедитесь, что на дисках нет важной информации.

Шаг 2: Создание и форматирование

  1. Просмотрите сводку: уровень RAID, количество дисков, расчётная ёмкость
  2. Нажмите "Создать"

  3. Откроется окно прогресса:

    • Создание mdadm-массива
    • Форматирование в файловую систему Btrfs
    • Монтирование массива
    • Добавление в автозагрузку (fstab)

  4. Дождитесь завершения всех этапов

!!! note "Примечание" Инициальная синхронизация RAID 5/6/10 может занять несколько часов в зависимости от размера дисков. Массив доступен для использования сразу, но производительность будет снижена до завершения синхронизации.

После создания

Созданный массив появится на странице "Диски и RAID" в виде карточки с информацией:

  • Уровень RAID и количество дисков
  • Текущее состояние (активен / синхронизируется)
  • Точка монтирования (например, /mnt/data)
  • Тип файловой системы (Btrfs)
  • Использованное и свободное пространство

Теперь вы можете:

  • Создать общие папки на этом массиве
  • Настроить снапшоты для защиты данных
  • Включить Guard для защиты от шифровальщиков

Состояния массива

Состояние Описание
Активен Все диски работают нормально
Синхронизация Идёт начальная синхронизация или перестройка. Отображается прогресс в процентах
Деградирован Один или несколько дисков вышли из строя. Массив работает, но без отказоустойчивости. Требуется замена диска
Неактивен Массив остановлен

11. Управление RAID-массивами

Управление RAID-массивами

После создания массива доступны операции по его обслуживанию: монтирование, расширение, апгрейд уровня, управление субтомами и удаление.

Карточка массива

Каждый массив отображается на странице "Диски и RAID" в виде карточки. На ней видно:

  • Имя массива (например, md127)
  • Уровень RAID и количество дисков
  • Состояние: активен, деградирован, синхронизируется
  • Точка монтирования и файловая система
  • Использованное пространство (полоска-прогресс)

Монтирование и размонтирование

Монтирование

Если массив не смонтирован (отображается "не смонтирован"):

  1. Нажмите кнопку "Смонтировать" на карточке массива
  2. Система автоматически определит файловую систему
  3. Массив будет смонтирован и добавлен в автозагрузку

!!! note "Примечание" Кнопка "Смонтировать" неактивна, если на массиве не обнаружена файловая система.

Размонтирование

  1. Нажмите кнопку "Размонтировать" на карточке
  2. Подтвердите действие в диалоговом окне
  3. Массив будет отключён от файловой системы

!!! warning "Внимание" Перед размонтированием убедитесь, что ни один сервис не использует данные на массиве. Активные SMB/NFS-сессии будут принудительно прерваны.

Расширение массива

Расширение позволяет добавить новый диск в существующий массив и увеличить его ёмкость.

  1. Нажмите кнопку "Расширить" на карточке массива
  2. Выберите свободный диск из списка
  3. Подтвердите действие
  4. Система выполнит:
    • Добавление диска в массив (mdadm --add)
    • Увеличение количества активных дисков (mdadm --grow)
    • Перестройку данных (reshape)
    • Автоматическое расширение файловой системы после завершения перестройки

!!! note "Примечание" Перестройка массива может занять несколько часов. Массив доступен во время этого процесса, но производительность снижена. Прогресс отображается на карточке.

Апгрейд уровня RAID (5 -> 6)

Если у вас массив RAID 5 и вы хотите повысить отказоустойчивость:

  1. Убедитесь, что массив в состоянии "Активен" (не деградирован)
  2. На карточке массива появится кнопка "Апгрейд RAID5 -> 6"
  3. Нажмите её
  4. В модальном окне выберите свободный диск для добавления
  5. Подтвердите операцию
  6. Система добавит диск и перестроит массив с RAID 5 на RAID 6

После апгрейда массив выдержит одновременный выход из строя двух дисков вместо одного.

!!! danger "Важно" Во время перестройки (reshape) НЕ ОТКЛЮЧАЙТЕ питание и НЕ перезагружайте сервер. Это может привести к потере данных. Используйте ИБП для защиты.

Субтома Btrfs

Для массивов с файловой системой Btrfs доступно управление субтомами (subvolumes). Субтома позволяют:

  • Создавать изолированные области данных
  • Настраивать независимые снапшоты для каждого субтома
  • Устанавливать квоты на объём данных

Просмотр субтомов

  1. На смонтированном Btrfs-массиве нажмите кнопку "Субтома"
  2. Откроется список существующих субтомов

Создание субтома

  1. Нажмите "+ Создать субтом"
  2. Введите имя субтома
  3. Нажмите "Создать"

Удаление субтома

  1. Найдите субтом в списке
  2. Нажмите "Удалить"
  3. Подтвердите действие

!!! warning "Внимание" Удаление субтома удаляет все данные внутри него. Перед удалением убедитесь, что на субтоме нет нужных файлов или есть актуальный снапшот.

Удаление массива

  1. Нажмите кнопку "Удалить" на карточке массива
  2. Откроется окно подтверждения со списком дисков-участников
  3. Подтвердите удаление
  4. Система выполнит:
    • Размонтирование массива
    • Очистку записи в автозагрузке
    • Остановку массива
    • Очистку RAID-суперблоков на дисках

!!! danger "Важно" Удаление массива уничтожает все данные на нём. Эта операция необратима. Перед удалением система также очистит связанные SMB/NFS-шары.

12. Замена диска в RAID

Замена диска в RAID

При выходе диска из строя или при предупреждениях S.M.A.R.T. необходимо заменить диск для восстановления отказоустойчивости массива. RusNAS поддерживает горячую замену без остановки массива.

Когда нужна замена

Замену диска следует выполнить, если:

  • Массив перешёл в состояние "Деградирован" -- один из дисков вышел из строя
  • S.M.A.R.T. показывает предупреждение или ошибку для диска
  • На дашборде появилось уведомление о проблемах с дисками

!!! danger "Важно" Деградированный массив RAID 5 работает без отказоустойчивости. Выход ещё одного диска приведёт к полной потере данных. Замените диск как можно скорее.

Проверка S.M.A.R.T.

На странице "Диски и RAID" для каждого физического диска отображается состояние S.M.A.R.T.:

Индикатор Значение
Здоров (зелёный) Диск работает нормально
Предупреждение (жёлтый) Обнаружены ранние признаки деградации. Рекомендуется замена
Ошибка (красный) Критическое состояние. Замена необходима

Нажмите на диск для просмотра детальной информации: модель, серийный номер, температура, количество переназначённых секторов.

Пошаговая замена

1. Определите неисправный диск

На карточке деградированного массива неисправный диск отображается красным цветом с пометкой "Отказ" или "Отсутствует".

Запомните серийный номер неисправного диска -- он понадобится для идентификации при физической замене.

2. Физическая замена (если поддерживается горячая замена)

  1. Извлеките неисправный диск из корпуса сервера
  2. Установите новый диск того же или большего размера
  3. На странице "Диски и RAID" нажмите "Сканировать SCSI" для обнаружения нового диска

3. Запуск перестройки

  1. На карточке деградированного массива нажмите "Заменить диск"
  2. В модальном окне:
    • Отображается информация о неисправном диске
    • Выберите новый диск из списка свободных
    • Для каждого диска показана информация S.M.A.R.T. и размер
  3. Подтвердите серийный номер -- система попросит ввести серийный номер неисправного диска для защиты от ошибки
  4. Нажмите "Заменить"

4. Мониторинг перестройки

После запуска замены начнётся перестройка (rebuild) массива:

  • На карточке массива отображается прогресс в процентах
  • Расчётное время завершения
  • Скорость перестройки

Во время перестройки:

  • Массив доступен для чтения и записи
  • Производительность снижена
  • Не отключайте питание -- используйте ИБП

!!! note "Примечание" Время перестройки зависит от размера массива. Для дисков объёмом 4 ТБ перестройка может занять 6-12 часов.

5. Завершение

После завершения перестройки:

  • Массив вернётся в состояние "Активен"
  • Отказоустойчивость восстановлена
  • Автообновление страницы каждые 4 секунды покажет финальный статус

Рекомендации

  • Всегда держите один запасной диск в резерве
  • Настройте уведомления по email или Telegram для оперативного оповещения о деградации
  • Рассмотрите апгрейд до RAID 6, если у вас RAID 5 с большими дисками -- вероятность второго отказа во время перестройки увеличивается с ростом ёмкости
  • Регулярно проверяйте S.M.A.R.T. и меняйте диски при появлении предупреждений, не дожидаясь отказа

!!! tip "Совет" Страница дисков автоматически обновляется каждые 4 секунды, пока массив находится в состоянии перестройки или деградации. Вам не нужно обновлять страницу вручную.

13. SSD-кеширование

SSD-кеширование

SSD-кеширование позволяет использовать быстрый SSD-диск для ускорения операций ввода-вывода на RAID-массиве из обычных жёстких дисков. Часто запрашиваемые данные копируются на SSD, обеспечивая значительный прирост скорости чтения.

Принцип работы

Технология dm-cache (LVM) создаёт кеширующий слой между приложениями и основным хранилищем:

  1. При первом чтении данных они копируются с HDD на SSD-кеш
  2. При повторном запросе данные отдаются с SSD (в десятки раз быстрее)
  3. Записи в зависимости от режима либо проходят через кеш, либо идут напрямую на HDD

Режимы кеширования

Режим Описание Безопасность Производительность
Writethrough Данные записываются одновременно на SSD и HDD. При отказе SSD данные не теряются Высокая Ускорение только чтения
Writeback Данные сначала записываются на SSD, затем фоново переносятся на HDD. Ускоряет и чтение, и запись Средняя -- при отказе SSD возможна потеря незаписанных данных Ускорение чтения и записи

!!! warning "Внимание" Режим Writeback обеспечивает лучшую производительность, но при внезапном отказе SSD (без корректного отключения) возможна потеря недавно записанных данных. Используйте ИБП для защиты.

Где найти

Настройки SSD-кеширования находятся на странице "Диски и RAID" в разделе "SSD-кеш" (панель под карточками массивов).

Добавление SSD-кеша

  1. Подключите SSD к серверу
  2. На странице "Диски и RAID" убедитесь, что SSD обнаружен в списке физических дисков
  3. В разделе "SSD-кеш" нажмите "+ Добавить кеш"
  4. Выберите параметры:
Поле Описание
SSD-диск Выберите SSD из списка свободных дисков
Целевой массив RAID-массив, для которого создаётся кеш
Режим Writethrough (рекомендуется) или Writeback
  1. Нажмите "Создать"

!!! note "Примечание" SSD будет полностью использован для кеширования. Все данные на нём будут удалены.

Мониторинг эффективности

После активации кеша в панели отображается:

Метрика Описание
Hit Rate Процент запросов, обслуженных из кеша. Чем выше, тем эффективнее кеш
Использование Какой процент кеша заполнен
Режим Текущий режим работы (writethrough/writeback)
Статус Активен / Неактивен

Хороший показатель Hit Rate -- выше 60%. Если Hit Rate ниже 30%, ваша рабочая нагрузка плохо кешируется (например, преимущественно линейная запись больших файлов).

Удаление SSD-кеша

  1. В разделе "SSD-кеш" нажмите "Удалить" рядом с кешем
  2. Подтвердите удаление
  3. Система безопасно отключит кеш:
    • Для writeback: сначала сбросит все кешированные данные на HDD
    • Затем отключит SSD от массива

!!! tip "Совет" SSD-кеш наиболее эффективен при смешанной нагрузке: множество мелких файлов (документы, виртуальные машины, базы данных). Для потоковой записи больших файлов (видеонаблюдение) кеш даёт минимальный эффект.

Рекомендации по выбору SSD

  • Ёмкость: 10-20% от объёма основного массива достаточно для большинства нагрузок
  • Тип: серверные SSD с высоким ресурсом записи (TBW) предпочтительны
  • Интерфейс: NVMe обеспечит лучшую производительность, SATA SSD тоже подойдёт

См. также: Создание массива | Оптимизация производительности

14. Резервный режим (Backup Mode)

Резервный режим (Backup Mode)

Резервный режим позволяет автоматически останавливать вращение жёстких дисков RAID-массива в период бездействия. Это снижает износ дисков, потребление энергии и шум -- идеально для массивов, используемых только для резервного копирования.

Когда использовать

  • RAID-массив используется для резервных копий и обращения к нему происходят редко (раз в сутки или реже)
  • Требуется снизить энергопотребление сервера
  • Требуется продлить срок службы жёстких дисков

Где найти

На странице "Диски и RAID" на карточке каждого массива есть кнопка "Резервный режим". Нажмите её для открытия панели настроек.

Настройка

Параметр Описание
Включить резервный режим Активирует автоматический спящий режим для дисков массива
Тайм-аут бездействия Время в минутах, после которого неиспользуемые диски будут остановлены. Рекомендуется: 15-30 минут

После включения и сохранения настроек система начнёт отслеживать активность дисков.

Состояния дисков

На карточке массива отображается текущее состояние:

Состояние Бейдж Описание
Активен Зелёный Диски вращаются, массив доступен
Спит Серый Диски остановлены, массив в спящем режиме
Пробуждение Синий Диски раскручиваются после запроса

Пробуждение и засыпание

Автоматическое пробуждение

При обращении к данным на массиве (через SMB, NFS, iSCSI или напрямую) диски автоматически раскрутятся. Первый запрос после пробуждения будет обработан с задержкой 5-15 секунд (время раскрутки дисков).

Ручное пробуждение

На панели резервного режима нажмите "Пробудить" для принудительной раскрутки дисков.

Ручное усыпление

Нажмите "Усыпить" для немедленной остановки дисков, не дожидаясь тайм-аута.

Мониторинг на дашборде

На дашборде карточка массива в резервном режиме показывает:

  • Текущее состояние (спит/активен)
  • Счётчик пробуждений -- сколько раз диски были разбужены
  • Время в спящем режиме

!!! warning "Внимание" Некоторые операции блокируются, когда массив находится в спящем режиме:

- Расширение массива
- Замена диска
- Размонтирование
- Управление субтомами

Система предупредит вас и предложит сначала пробудить массив.

Взаимодействие с SSD-кешем

!!! danger "Важно" Если на массиве включён SSD-кеш в режиме writeback, будьте осторожны с резервным режимом. SSD-кеш может содержать незаписанные данные. Система автоматически сбрасывает кеш на диски перед усыплением, но при аварийном отключении возможна потеря данных.

Рекомендуется использовать резервный режим с SSD-кешем только в режиме writethrough.

Рекомендации

  • Устанавливайте тайм-аут не менее 15 минут -- слишком частые циклы пуска/остановки вредят дискам больше, чем постоянная работа
  • Для массивов резервного копирования оптимальный тайм-аут: 30-60 минут
  • Избегайте резервного режима для массивов с активным SMB/NFS-доступом -- постоянные пробуждения сведут экономию к нулю

См. также: Управление массивами | SSD-кеширование | ИБП

15. Guard: защита от шифровальщиков

Guard: защита от шифровальщиков

Guard -- встроенная система защиты RusNAS от программ-шифровальщиков (ransomware). Guard отслеживает подозрительную активность на файловых шарах и может автоматически блокировать атаку, сохраняя данные.

Как работает Guard

Guard работает в фоновом режиме и непрерывно анализирует файловые операции на защищаемых директориях. При обнаружении подозрительной активности Guard:

  1. Фиксирует событие в журнале
  2. В зависимости от режима: уведомляет администратора или блокирует источник атаки
  3. Автоматически создаёт снапшот данных для восстановления

Четыре метода обнаружения

Guard использует несколько независимых методов для максимальной надёжности:

1. Ловушки (Honeypot)

Система размещает специальные файлы-приманки в защищаемых директориях. Эти файлы невидимы для обычных пользователей, но шифровальщики пытаются зашифровать всё подряд. Изменение файла-приманки -- верный признак атаки.

2. Анализ энтропии

Шифровальщики превращают обычные файлы (документы, фото) в "шум" -- данные с высокой энтропией (случайностью). Guard измеряет энтропию изменяемых файлов. Если файл после изменения стал похож на случайный набор байтов -- это подозрительно.

3. Мониторинг IOPS

Шифровальщики обычно шифруют тысячи файлов за короткое время, создавая аномальный всплеск операций ввода-вывода. Guard отслеживает частоту файловых операций и реагирует на необычно высокую активность.

4. Контроль расширений

Guard содержит базу из ~200 расширений, характерных для зашифрованных файлов (.encrypted, .locked, .cry и т.д.). Появление файлов с такими расширениями -- сигнал тревоги.

Три режима реагирования

Режим Описание Когда использовать
Мониторинг Guard фиксирует события, но не предпринимает активных действий. Уведомляет администратора Для наблюдения, тестирования, минимизации ложных срабатываний
Активный Guard блокирует подозрительный источник (SMB-сессию) и создаёт снапшот Рекомендуемый режим для рабочей среды
Супер-безопасный Максимально агрессивное реагирование: мгновенная блокировка при первом подозрении Для хранилищ с критически важными данными

!!! tip "Совет" Рекомендуется начать с режима Мониторинг на 1-2 недели, чтобы убедиться в отсутствии ложных срабатываний, затем переключить в Активный режим.

PIN-код Guard

Guard защищён собственным PIN-кодом, независимым от пароля учётной записи. PIN требуется для:

  • Изменения режима работы
  • Отключения Guard
  • Разблокировки заблокированных пользователей
  • Изменения настроек обнаружения

Первоначальная установка PIN

При первом открытии страницы Guard система попросит задать PIN-код. Выберите надёжный код и запомните его.

Восстановление PIN

Если PIN утерян, его можно сбросить только через SSH-доступ к серверу:

sudo rusnas-guard --reset-pin

!!! warning "Внимание" Разделение PIN-кода Guard и пароля администратора сделано намеренно. Даже если злоумышленник получит доступ к веб-интерфейсу, он не сможет отключить защиту без PIN.

Где найти

Страница Guard доступна в боковой панели. Она содержит три вкладки:

  • Обзор -- текущий статус, статистика, инфографика
  • Журнал событий -- подробный лог обнаруженных угроз
  • Настройки -- конфигурация методов обнаружения и режимов

Карточка Guard на дашборде

На главной странице (Дашборд) карточка Guard показывает:

  • Текущий режим работы
  • Количество отслеживаемых директорий
  • Количество обнаруженных событий за сутки
  • Общий статус: защищён / требует внимания

16. Настройки Guard

Настройки Guard

Страница настроек Guard позволяет управлять методами обнаружения, защищаемыми директориями и параметрами реагирования.

Как открыть

Откройте страницу Guard и перейдите на вкладку "Настройки". Для изменения настроек потребуется ввести PIN-код Guard.

Методы обнаружения

Каждый метод можно включить или отключить независимо:

Метод Переключатель Рекомендация
Ловушки (Honeypot) Вкл/Выкл Оставить включённым. Самый надёжный метод с минимумом ложных срабатываний
Анализ энтропии Вкл/Выкл Включить. Может давать редкие ложные срабатывания на архивах и мультимедиа
Мониторинг IOPS Вкл/Выкл Включить. Отключить, если на сервере регулярно выполняются массовые операции с файлами
Контроль расширений Вкл/Выкл Включить. Дополнительный слой защиты

Порог энтропии

Для метода анализа энтропии можно настроить порог срабатывания:

  • Значение по умолчанию: 7.5 (из 8.0)
  • Ниже = чувствительнее: больше срабатываний, включая ложные
  • Выше = менее чувствительно: меньше ложных срабатываний, но может пропустить атаку

!!! tip "Совет" Для большинства рабочих нагрузок значение 7.5 оптимально. Если вы работаете с большим количеством сжатых файлов (архивы, видео), увеличьте порог до 7.8.

Защищаемые директории

Раздел отображает список директорий, которые Guard мониторит.

Добавление директории

  1. Нажмите "+ Добавить путь"
  2. Введите полный путь к директории (например, /mnt/data/public)
  3. Нажмите "Сохранить"

Удаление директории

  1. Найдите директорию в списке
  2. Нажмите "Удалить" рядом с ней
  3. Подтвердите действие

!!! note "Примечание" Рекомендуется мониторить директории, доступные по сети (SMB/NFS-шары). Именно через них чаще всего происходят атаки шифровальщиков.

Скрытие ловушек из SMB

По умолчанию файлы-ловушки (honeypot) скрыты от пользователей при просмотре SMB-шар. Этот параметр управляется переключателем:

  • Скрывать ловушки из SMB (рекомендуется) -- пользователи не видят служебные файлы
  • Показывать ловушки -- файлы видны, что может вызвать вопросы пользователей

!!! warning "Внимание" Если ловушки видны пользователям, они могут случайно их удалить или изменить, что вызовет ложное срабатывание.

Настройка снапшотов при атаке

Можно настроить автоматическое создание снапшота при обнаружении атаки:

  • Авто-снапшот при атаке: Вкл/Выкл
  • При включении Guard создаст снапшот защищаемого тома в момент обнаружения угрозы

Это позволяет восстановить данные до состояния непосредственно перед атакой.

Смена PIN-кода

  1. В настройках Guard нажмите "Сменить PIN"
  2. Введите текущий PIN
  3. Введите новый PIN дважды
  4. Нажмите "Сохранить"

!!! danger "Важно" Запишите новый PIN в безопасном месте. Восстановление PIN через веб-интерфейс невозможно. Сброс возможен только через SSH: sudo rusnas-guard --reset-pin.

Управление блокировками

В активном и супер-безопасном режимах Guard может заблокировать SMB-сессию подозрительного пользователя. Для разблокировки:

  1. Перейдите на вкладку "Журнал событий"
  2. Найдите событие блокировки
  3. Нажмите "Разблокировать"
  4. Введите PIN-код Guard

Запуск и остановка детектора

Кнопки "Запустить" и "Остановить" управляют модулем обнаружения угроз. Фоновый сервис Guard всегда работает -- кнопки влияют только на активный мониторинг.

  • Запустить -- Guard начинает отслеживать файловые операции
  • Остановить -- мониторинг приостанавливается (журнал и настройки доступны)

Типичные сценарии

Ложное срабатывание по энтропии

Некоторые легитимные файлы имеют высокую энтропию: архивы (ZIP, RAR, 7z), зашифрованные бэкапы, мультимедиа. Если вы видите частые ложные срабатывания:

  • Проверьте, какие файлы вызывают срабатывание
  • При необходимости увеличьте порог энтропии в настройках
  • Или исключите конкретные директории из мониторинга

При обнаружении реальной атаки:

  1. Guard заблокирует источник (в активном режиме)
  2. Автоматический снапшот будет создан (если включён)
  3. Изолируйте заражённый компьютер от сети
  4. Восстановите данные из снапшота
  5. После очистки компьютера разблокируйте пользователя

18. Управление снапшотами

Управление снапшотами

Снапшоты (снимки) -- мгновенные копии состояния данных на определённый момент времени. RusNAS использует технологию Btrfs для создания снапшотов за секунды, без прерывания работы сервера.

Что такое снапшот

Снапшот фиксирует состояние файлов и папок на конкретный момент. После создания снапшота:

  • Все текущие файлы сохраняются в их текущем виде
  • Дальнейшие изменения файлов не затрагивают снапшот
  • Снапшот можно использовать для восстановления данных

Снапшоты не являются полной копией данных -- они используют принцип "копирование при записи" (copy-on-write) и занимают минимум места.

Где найти

Откройте страницу Снапшоты в боковой панели. Первая вкладка "Снапшоты" отображает список всех созданных снимков.

Список снапшотов

Таблица снапшотов содержит:

Столбец Описание
Имя Автоматически сгенерированное имя с датой и временем
Субтом Субтом Btrfs, для которого создан снапшот
Тип Ручной / По расписанию / Перед обновлением / Guard
Дата создания Дата и время создания
Заблокирован Защищён от удаления (значок замка)
Действия Кнопки управления

  1. Нажмите кнопку "+ Создать снапшот"
  2. В модальном окне выберите:
Поле Описание
Субтом Выберите субтом для снимка из выпадающего списка
Комментарий Необязательное описание (например, "Перед миграцией")
  1. Нажмите "Создать"

Снапшот создаётся мгновенно (менее секунды) и появится в таблице.

Просмотр содержимого

Для просмотра файлов внутри снапшота:

  1. Найдите нужный снапшот в таблице
  2. Нажмите "Обзор" или значок папки
  3. Откроется файловый браузер, показывающий содержимое снапшота на момент его создания

Это позволяет найти нужную версию файла перед восстановлением.

Восстановление данных

Для восстановления файла или папки из снапшота:

  1. Найдите нужный снапшот в таблице
  2. Нажмите "Обзор" для просмотра содержимого
  3. Найдите нужный файл или папку
  4. Скопируйте файл из снапшота в рабочую директорию

!!! note "Примечание" Снапшоты доступны только для чтения. Вы не можете изменять файлы внутри снапшота -- только копировать их обратно в рабочее пространство.

Блокировка и разблокировка

Заблокированный снапшот защищён от автоматического и ручного удаления:

Блокировка

  1. Найдите снапшот в таблице
  2. Нажмите значок замка или кнопку "Заблокировать"
  3. Снапшот помечается значком замка

Разблокировка

  1. Нажмите значок замка на заблокированном снапшоте
  2. Подтвердите разблокировку

!!! tip "Совет" Блокируйте важные снапшоты (например, перед крупным обновлением), чтобы политика хранения не удалила их автоматически.

Удаление снапшота

  1. Найдите снапшот в таблице
  2. Нажмите кнопку "Удалить" (значок корзины)
  3. Подтвердите удаление

!!! warning "Внимание" Удаление снапшота необратимо. Заблокированные снапшоты не могут быть удалены -- сначала разблокируйте их.

Типы снапшотов

Тип Описание
Ручной Создан администратором вручную
По расписанию Создан автоматически по расписанию
Перед обновлением Создан автоматически перед apt upgrade
Guard Создан автоматически при обнаружении угрозы Guard

Сколько места занимают снапшоты

Снапшоты используют copy-on-write и занимают место только для изменённых данных:

  • Сразу после создания снапшот практически не занимает места
  • Со временем, по мере изменения файлов, размер растёт
  • Чем больше файлов изменено с момента снапшота, тем больше он занимает

Для оценки используемого пространства откройте Анализатор пространства.

  • За последнюю неделю -- с точностью до дня

!!! note "Примечание" Удаление расписания не удаляет уже созданные снапшоты. Они останутся в списке на вкладке "Снапшоты" и будут доступны для восстановления.

Снапшоты перед обновлением

RusNAS автоматически создаёт снапшот перед каждым системным обновлением (apt upgrade). Эта функция работает независимо от расписания и не требует настройки.

Если обновление вызвало проблему, вы сможете восстановить данные из этого снапшота.

Рекомендации

  • Создайте хотя бы одно расписание для каждого субтома с данными
  • Настройте политику хранения с учётом доступного места на диске
  • Заблокируйте важные снапшоты, чтобы политика хранения не удалила их
  • Регулярно проверяйте, что снапшоты создаются (дата последнего снапшота в таблице)

См. также: Управление снапшотами | Репликация

20. Репликация снапшотов

Репликация снапшотов

Репликация позволяет автоматически копировать снапшоты на удалённый сервер по SSH. Это обеспечивает защиту данных от аппаратных сбоев, краж и стихийных бедствий -- даже при полном выходе из строя основного сервера данные можно восстановить с удалённой копии.

Принцип работы

  1. RusNAS создаёт Btrfs-снапшот на локальном сервере
  2. Снапшот передаётся через SSH на удалённый сервер с помощью btrfs send/receive
  3. Первая передача -- полная копия всех данных
  4. Последующие передачи -- только изменения (инкрементальные), что значительно экономит время и трафик

Где найти

Откройте страницу Снапшоты и перейдите на вкладку "Репликация".

Подготовка

Требования к удалённому серверу

  • Операционная система Linux с поддержкой Btrfs
  • Смонтированный том Btrfs для приёма снапшотов
  • SSH-доступ (рекомендуется по ключу, без пароля)
  • Достаточно свободного пространства

Настройка SSH-ключей

Для автоматической репликации необходимо настроить SSH-доступ без пароля:

  1. На сервере RusNAS сгенерируйте SSH-ключ (если ещё не создан)
  2. Скопируйте публичный ключ на удалённый сервер
  3. Проверьте, что подключение по SSH работает без запроса пароля

!!! warning "Внимание" Настройка SSH-ключей выполняется через командную строку (SSH-доступ к серверу RusNAS). Веб-интерфейс позволяет проверить доступность удалённого сервера, но не настраивает ключи.

Создание задачи репликации

  1. Нажмите "+ Добавить задачу"
  2. Заполните параметры:
Поле Описание
Субтом Локальный субтом Btrfs для репликации
Удалённый хост IP-адрес или имя удалённого сервера
Пользователь Имя пользователя для SSH-подключения
Путь на удалённом сервере Директория на удалённом Btrfs-томе для приёма снапшотов
  1. Нажмите "Проверить соединение" для проверки доступности
  2. Если проверка успешна, нажмите "Сохранить"

Список задач

Таблица отображает все настроенные задачи репликации:

Столбец Описание
Субтом Что реплицируется
Назначение Куда (хост + путь)
Последний запуск Дата и результат последней репликации
Статус Успешно / Ошибка / В процессе
Действия Запуск, редактирование, удаление

Ручной запуск

  1. Найдите задачу в таблице
  2. Нажмите "Запустить"
  3. Прогресс отображается в строке задачи

Автоматический запуск

Репликация запускается автоматически по таймеру (каждые 5 минут проверяется, есть ли новые снапшоты для передачи). Если с момента последней репликации появились новые снапшоты, они будут переданы.

Полная и инкрементальная передача

Тип Когда Что передаётся Время
Полная Первый запуск задачи Весь снапшот целиком Зависит от объёма данных и скорости сети
Инкрементальная Все последующие Только изменения с момента последнего переданного снапшота Значительно быстрее

!!! note "Примечание" Инкрементальная передача работает автоматически. Система сравнивает последний переданный снапшот с текущим и передаёт только разницу.

Мониторинг

При активной передаче данных:

  • В столбце "Статус" отображается "В процессе"
  • Отображается объём переданных данных
  • После завершения -- результат (успех или ошибка)

Удаление задачи

  1. Нажмите "Удалить" рядом с задачей
  2. Подтвердите действие

!!! note "Примечание" Удаление задачи не удаляет уже переданные снапшоты с удалённого сервера. Для их очистки подключитесь к удалённому серверу напрямую.

Рекомендации

  • Используйте гигабитное или более быстрое сетевое подключение
  • Для репликации через интернет настройте VPN-туннель для защиты трафика
  • Регулярно проверяйте доступность удалённого сервера
  • Убедитесь, что на удалённом сервере достаточно свободного места

21. Дедупликация

Дедупликация

Дедупликация находит одинаковые блоки данных в файлах и заменяет дубликаты ссылками на единственную копию. Это позволяет значительно сэкономить место на диске, особенно при хранении множества похожих файлов (виртуальные машины, бэкапы, проекты).

Принцип работы

RusNAS использует утилиту duperemove совместно с технологией reflink файловой системы Btrfs:

  1. Duperemove сканирует файлы и находит одинаковые блоки данных
  2. Дубликаты заменяются reflink-ссылками на единственный экземпляр
  3. Файлы продолжают выглядеть и работать как независимые
  4. При изменении одного из файлов для него создаётся отдельная копия (copy-on-write)

Где найти

Откройте страницу Дедупликация в боковой панели.

Текущее состояние

Главная страница дедупликации показывает:

Показатель Описание
Статус Работает / Ожидание / Не запускалась
Последний запуск Дата и время последнего анализа
Экономия Объём сэкономленного пространства (shared extents)
Обработано файлов Количество файлов, проанализированных при последнем запуске

Ручной запуск

  1. Нажмите кнопку "Запустить дедупликацию"
  2. Система запустит анализ и дедупликацию выбранных томов
  3. Прогресс отображается на странице
  4. По завершении обновится статистика экономии

!!! note "Примечание" Дедупликация может занять от нескольких минут до нескольких часов в зависимости от объёма данных. Процесс работает с пониженным приоритетом (Nice=19, idle IO) и не влияет на производительность сервера.

Настройка расписания

Для автоматической дедупликации по расписанию:

  1. Нажмите "Настройки" или перейдите в раздел расписания
  2. Включите переключатель "Расписание дедупликации"
  3. Задайте cron-расписание (например, каждую ночь в 3:00)
  4. Нажмите "Сохранить"

Рекомендуемое расписание: раз в сутки в ночное время.

Выбор томов

В настройках можно указать, какие тома анализировать:

  1. Откройте "Настройки"
  2. В разделе "Тома" отметьте нужные тома
  3. Нажмите "Сохранить"

!!! tip "Совет" Дедупликация наиболее эффективна для томов с множеством похожих файлов: репозитории виртуальных машин, бэкапы, общие папки с документами.

История экономии

В нижней части страницы отображается график экономии за последние 7 дней. По графику видно, как меняется объём сэкономленного пространства после каждого запуска.

Влияние на производительность

Дедупликация спроектирована с минимальным влиянием на основную работу сервера:

  • Приоритет CPU: Nice=19 (самый низкий)
  • Приоритет ввода-вывода: Idle (только свободные ресурсы)
  • Работающие сервисы (SMB, NFS, iSCSI) не замедляются

!!! warning "Внимание" Хотя дедупликация безопасна и работает с низким приоритетом, первый запуск на большом томе может создать заметную нагрузку на дисковую подсистему. Рекомендуется первый запуск выполнить в нерабочее время.

Когда дедупликация эффективна

Сценарий Ожидаемая экономия
Виртуальные машины (много одинаковых ОС) 30-70%
Бэкапы (инкрементальные) 20-50%
Домашние папки пользователей 10-30%
Медиа-файлы (фото, видео) 0-5% (уже сжаты)
Уникальные данные Минимальная

22. ИБП: подключение и настройка

ИБП: подключение и настройка

Источник бесперебойного питания (ИБП / UPS) защищает сервер от потери данных при отключении электричества. RusNAS управляет ИБП через систему NUT (Network UPS Tools) и может автоматически корректно завершить работу при разряде батареи.

Поддерживаемые подключения

Тип Описание Кабель
USB Прямое подключение ИБП к серверу через USB USB A-B (обычно идёт в комплекте с ИБП)
SNMP Сетевое управление через протокол SNMP Ethernet (ИБП с сетевой картой)
Сетевой клиент Подключение к NUT-серверу другого компьютера Ethernet (если ИБП подключён к другому серверу)

Где найти

Откройте страницу ИБП в боковой панели.

Автоматическое обнаружение

При первом открытии страницы ИБП система автоматически сканирует подключённые устройства:

  1. Если ИБП обнаружен по USB, его модель и параметры отобразятся на странице
  2. Если ИБП не обнаружен, нажмите "Сканировать устройства" для повторного поиска

!!! note "Примечание" Для обнаружения ИБП по USB кабель должен быть подключён физически, а ИБП включён.

Режимы работы NUT

При настройке ИБП выберите режим работы:

Режим Описание Когда использовать
Standalone Сервер подключён напрямую к ИБП и управляет только собой Один сервер + один ИБП
Netserver Сервер подключён к ИБП и делится информацией с другими серверами по сети Несколько серверов на одном ИБП
Netclient Сервер получает информацию об ИБП от другого NUT-сервера по сети Вторичный сервер на общем ИБП

Для большинства установок подходит режим Standalone.

Пошаговая настройка (USB)

  1. Подключите ИБП к серверу USB-кабелем
  2. Откройте страницу ИБП
  3. Нажмите "Сканировать устройства" (если ИБП не обнаружен автоматически)
  4. После обнаружения нажмите "Настроить"
  5. Выберите режим: Standalone
  6. Проверьте параметры:
Параметр Описание
Драйвер Автоматически определяется при сканировании
Порт USB-порт устройства (auto для USB-подключения)
Имя Название ИБП в системе (произвольное, латиница)
  1. Нажмите "Сохранить"
  2. Система запустит NUT-сервис и начнёт мониторинг

Настройка сетевого клиента

Если ИБП подключён к другому серверу с NUT:

  1. Выберите режим Netclient
  2. Укажите параметры:
Параметр Описание
Адрес NUT-сервера IP-адрес сервера, к которому подключён ИБП
Имя ИБП Имя ИБП на удалённом NUT-сервере
Логин Имя пользователя для подключения к NUT
Пароль Пароль для подключения
  1. Нажмите "Сохранить"

Перезапуск службы NUT

Если после изменения конфигурации мониторинг не работает:

  1. Нажмите "Перезапустить NUT" на странице ИБП
  2. Дождитесь перезапуска (несколько секунд)
  3. Статус ИБП должен обновиться

!!! warning "Внимание" Не все модели ИБП поддерживаются NUT. Если ваш ИБП не обнаруживается, проверьте совместимость на сайте NUT (networkupstools.org).

Следующий шаг: Мониторинг ИБП

23. ИБП: мониторинг и автоотключение

ИБП: мониторинг и автоотключение

После подключения и настройки ИБП страница мониторинга показывает текущее состояние питания и позволяет настроить автоматическое отключение сервера при разряде батареи.

Панель состояния

Показатель Описание

Оставшееся время Расчётное время автономной работы в минутах
Нагрузка Текущая нагрузка на ИБП в процентах от максимальной
Входное напряжение Напряжение электросети, В
Выходное напряжение Напряжение на выходе ИБП, В
Температура Внутренняя температура ИБП (если поддерживается)

Цветовые индикаторы статуса

Статус ИБП отображается цветным индикатором:

Цвет Статус Описание
Зелёный OL (On Line) Питание от сети, всё в порядке
Жёлтый OB (On Battery) Работа от батареи, электросеть отключена
Красный OB LB (Low Battery) Батарея разряжается, критический уровень

  • Текущий статус (цветной индикатор)
  • Уровень заряда
  • Нагрузку

Данные обновляются каждые 30 секунд.

Настройка автоматического отключения

RusNAS может автоматически завершить работу сервера при критическом разряде батареи, предотвращая потерю данных:

  1. На странице ИБП найдите раздел "Авто-отключение"
  2. Настройте параметры:
Параметр Описание Рекомендация
Порог заряда При каком уровне заряда начать отключение 20%
Порог времени При каком оставшемся времени автономной работы начать отключение 5 минут
Задержка Время в секундах между сигналом и отключением (для завершения открытых файлов) 30 секунд
  1. Нажмите "Сохранить"

!!! warning "Внимание" Убедитесь, что пороги автоотключения установлены с запасом. Сервер должен иметь достаточно времени для корректного завершения всех сервисов и сброса данных на диски.

Последовательность автоотключения

При срабатывании порогов система выполняет:

  1. Отправка уведомления администратору (email/Telegram, если настроены)
  2. Закрытие SMB/NFS-сессий
  3. Сброс кеша файловой системы на диски
  4. Размонтирование томов
  5. Корректное завершение работы ОС

Уведомления

RusNAS может отправлять уведомления при следующих событиях:

Событие Описание
Переход на батарею Электросеть отключена, ИБП работает от батареи
Возврат на сеть Электричество восстановлено
Низкий заряд Уровень заряда достиг критического порога
Замена батареи ИБП сообщает об износе батареи
Потеря связи Нет связи с ИБП

Журнал событий

Для просмотра истории событий ИБП:

  1. На странице ИБП нажмите "Журнал"
  2. Отобразится список событий: переключения питания, ошибки, уведомления

!!! tip "Совет" Регулярно проверяйте состояние ИБП. Если индикатор показывает RB (Replace Battery), замените батарею как можно скорее -- старая батарея может не обеспечить достаточного времени для корректного отключения.

Тестирование

Для проверки работоспособности автоотключения:

  1. Убедитесь, что все данные сохранены
  2. Отключите ИБП от электросети (выдерните кабель питания ИБП)
  3. Наблюдайте за страницей мониторинга -- статус должен переключиться на OB
  4. Подключите ИБП обратно до срабатывания порога автоотключения

26. Сеть: диагностика

Сеть: диагностика

Встроенные инструменты диагностики позволяют проверить сетевое подключение, разрешение DNS и доступность сервисов прямо из веб-интерфейса.

Где найти

На странице Сеть перейдите в раздел "Диагностика" или на соответствующую вкладку.

Ping

Проверка доступности хоста и измерение задержки.

  1. Введите IP-адрес или доменное имя (например, 8.8.8.8 или ya.ru)
  2. Нажмите "Ping"
  3. Результат покажет:
    • Доступен ли хост
    • Время отклика (мс)
    • Потерю пакетов (%)

Типичные сценарии

Результат Значение
Ответ за 1-10 мс Нормально для локальной сети
Ответ за 10-100 мс Нормально для интернет-ресурсов
Потеря пакетов > 0% Проблемы с сетью или перегрузка
Хост недоступен Нет связи, проверьте маршрутизацию и кабели

Traceroute

Показывает маршрут пакета от сервера до указанного хоста, включая все промежуточные маршрутизаторы.

  1. Введите IP-адрес или доменное имя
  2. Нажмите "Traceroute"
  3. Результат покажет список узлов на маршруте с задержками

!!! tip "Совет" Traceroute помогает определить, на каком участке сети возникает проблема. Если маршрут обрывается на определённом узле -- проблема на этом участке.

DNS Lookup

Проверка разрешения доменного имени в IP-адрес.

  1. Введите доменное имя (например, rusnas.ru)
  2. Нажмите "DNS Lookup"
  3. Результат покажет:
    • IP-адреса, связанные с доменом (A-записи)
    • Сервер имён, который дал ответ
    • Время ответа

Когда использовать

  • Проверка, что DNS-серверы работают
  • Проверка, что доменное имя разрешается правильно
  • Диагностика проблем с доступом к веб-сайтам

Port Check (проверка порта)

Проверка доступности конкретного TCP-порта на удалённом хосте.

  1. Введите IP-адрес или доменное имя
  2. Укажите номер порта (например, 80, 443, 22)
  3. Нажмите "Проверить порт"
  4. Результат: порт открыт или закрыт

Типичные порты для проверки

Порт Сервис
22 SSH
80 HTTP
443 HTTPS
445 SMB
2049 NFS
3260 iSCSI
9090 Cockpit (веб-интерфейс RusNAS)

Wake-on-LAN

Удалённое включение компьютера по сети через отправку специального "магического пакета".

  1. Введите MAC-адрес устройства (формат: AA:BB:CC:DD:EE:FF)
  2. Нажмите "Отправить WOL"
  3. Если устройство поддерживает Wake-on-LAN и настроено в BIOS, оно включится

!!! note "Примечание" Wake-on-LAN работает только в пределах одной локальной подсети. Устройство должно поддерживать WOL и быть настроено для приёма магических пакетов в BIOS/UEFI.

Рекомендации по диагностике

Нет доступа к интернету:

  1. Ping шлюза (маршрутизатора) -- проверка локальной связи
  2. Ping 8.8.8.8 -- проверка доступа к интернету по IP
  3. DNS Lookup ya.ru -- проверка DNS
  4. Если ping по IP работает, а DNS нет -- проблема в настройках DNS

Нет доступа к сетевой шаре:

  1. Ping клиентского компьютера -- базовая связь
  2. Port Check на 445 (SMB) или 2049 (NFS) -- доступность сервиса
  3. Проверить, что сервис запущен (индикатор на странице Хранилище)

28. Оптимизация производительности

Оптимизация производительности

Performance Tuner -- модуль автоматической оптимизации дисковой подсистемы, сети и системных параметров RusNAS. Система анализирует аппаратную конфигурацию и предлагает оптимальные настройки для вашего сценария использования.

Где найти

Откройте страницу Оптимизация в боковой панели.

Принцип работы

  1. Performance Tuner анализирует аппаратное обеспечение: CPU, RAM, диски, сетевые адаптеры
  2. Определяет текущий профиль нагрузки (файловый сервер, база данных, виртуализация)
  3. Предлагает оптимальные значения для 12 категорий параметров
  4. Позволяет применить настройки по одной или все сразу
  5. Поддерживает откат любого изменения

Профили нагрузки

При первом открытии система определяет профиль автоматически. Вы можете выбрать другой:

Профиль Описание Оптимизация
Файловый сервер SMB/NFS-шары для офисной работы Баланс чтения и записи
Медиа-хранилище Хранение и стриминг видео/аудио Большие буферы, потоковое чтение
Виртуализация Хранилище для ВМ (iSCSI/NFS) Низкая задержка, высокий IOPS
Резервное копирование Бэкапы и архивы Максимальная пропускная способность записи

12 категорий оптимизации

Память и подкачка (VM)

Параметр Описание
Swappiness Склонность к использованию подкачки (0-100). Низкое значение = меньше подкачки
Dirty ratio Процент RAM для буферизации записи перед сбросом на диск
Dirty background ratio Порог фоновой записи
VFS cache pressure Приоритет кеширования метаданных файловой системы

Планировщик ввода-вывода

Выбор планировщика: mq-deadline (рекомендуется для HDD), none/noop (для SSD), bfq (для интерактивных нагрузок).

Read-ahead

Размер буфера опережающего чтения. Увеличение ускоряет последовательное чтение, но может замедлить случайный доступ.

Параметры RAID

Размер кеша полосок (stripe cache) для mdadm-массивов. Влияет на производительность RAID 5/6.

Параметры Btrfs

Опции монтирования файловой системы: compress, noatime, space_cache и другие.

Сетевые параметры

Буферы TCP, размеры окна, настройки backlog.

Параметры NIC

Настройки сетевого адаптера через ethtool: offloading, ring buffers.

Samba

Параметры производительности SMB-сервера: socket options, use sendfile, aio read size.

NFS

Количество потоков nfsd и параметры кеширования.

CPU Governor

Режим управления частотой процессора: performance (максимальная частота), powersave (экономия), ondemand (авто).

IRQ Balancing

Распределение прерываний по ядрам процессора для оптимизации нагрузки.

TuneD

Интеграция с системным профилем TuneD.

Применение оптимизаций

  2. Для каждого параметра отображается:

31. Пользователи и группы

Пользователи и группы

Управление учётными записями пользователей и группами -- основа контроля доступа к данным на RusNAS. Каждый пользователь получает собственные учётные данные для подключения к общим папкам по SMB, FTP и другим протоколам.

Где найти

Действия Редактирование, смена пароля, удаление

Создание пользователя

  1. Нажмите кнопку "+ Создать пользователя"
  2. Заполните форму:
Поле Описание Обязательное
Имя пользователя Логин (латиница, без пробелов, строчные буквы) Да

Группы Выберите группы, в которые добавить пользователя Нет

Смена пароля

  1. Найдите пользователя в таблице
  2. Нажмите "Сменить пароль" (значок ключа)
  3. Введите новый пароль дважды
  4. Нажмите "Сохранить"

Пароль изменяется одновременно в системе и в Samba.

Редактирование пользователя

  1. Нажмите "Редактировать" рядом с пользователем
  2. Измените нужные параметры (группы, домашнюю папку)
  3. Нажмите "Сохранить"

Удаление пользователя

  1. Нажмите "Удалить" рядом с пользователем
  2. Система спросит, удалять ли домашнюю папку пользователя
  3. Подтвердите удаление

!!! warning "Внимание" При удалении пользователя он теряет доступ ко всем общим папкам. Файлы, принадлежащие пользователю в общих папках, сохраняются, но право собственности может потребовать перенастройки.

Группы

Группы упрощают управление доступом: вместо назначения прав каждому пользователю отдельно, вы назначаете права группе, и все её участники получают эти права.

Создание группы

  1. Перейдите в раздел "Группы" на странице пользователей
  2. Нажмите "+ Создать группу"
  3. Введите имя группы (латиница, без пробелов)
  4. Нажмите "Создать"

Добавление пользователя в группу

  1. Редактируйте пользователя
  2. В поле "Группы" отметьте нужные группы
  3. Нажмите "Сохранить"

Или:

  1. Редактируйте группу
  2. В списке участников отметьте нужных пользователей
  3. Нажмите "Сохранить"

Удаление группы

  1. Нажмите "Удалить" рядом с группой
  2. Подтвердите действие

!!! note "Примечание" Удаление группы не удаляет пользователей -- они просто перестают быть участниками этой группы.

Связь с общими папками

При создании SMB-шары вы можете указать допустимых пользователей или группы. Только они смогут подключиться к шаре.

Пример настройки доступа:

  • Шара public -- доступ для всех (гостевой или без ограничений)
  • Шара finance -- доступ только для группы accounting
  • Шара personal -- доступ только для пользователя director

Рекомендации

  • Создавайте именные учётные записи для каждого сотрудника -- это позволяет отслеживать, кто и когда обращался к файлам
  • Используйте группы для управления доступом к шарам
  • Устанавливайте сложные пароли (минимум 8 символов, буквы и цифры)
  • Регулярно проверяйте список пользователей и удаляйте учётные записи уволенных сотрудников
