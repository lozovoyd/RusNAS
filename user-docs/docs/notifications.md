# Уведомления

Система уведомлений RusNAS позволяет получать оповещения о событиях на NAS через 5 каналов: **Email**, **Telegram**, **MAX**, **SNMP** и **Webhook**.

Уведомления отправляются автоматически при:

- Обнаружении атаки шифровальщика (Guard)
- Деградации RAID-массива или отказе диска
- Переключении ИБП на батарею
- Ошибках снапшотов и репликации
- Проблемах с сетью
- Событиях контейнеров и других модулей

---

## Каналы доставки

### Email (SMTP)

1. Откройте страницу **Уведомления** в боковой панели
2. В секции **Email** включите переключатель
3. Заполните параметры SMTP-сервера:
    - **Хост** -- адрес SMTP-сервера (например, `smtp.yandex.ru`)
    - **Порт** -- 587 (STARTTLS) или 465 (SSL)
    - **TLS** -- выберите `starttls` или `ssl`
    - **Логин** и **Пароль** -- учётные данные SMTP
    - **Отправитель** -- адрес "от кого" (например, `rusNAS <noreply@example.com>`)
    - **Получатели** -- один или несколько email-адресов
4. Нажмите **Тест** для проверки -- на указанные адреса придёт тестовое письмо
5. Нажмите **Сохранить**

### Telegram

1. Создайте бота через [@BotFather](https://t.me/BotFather) и скопируйте токен
2. Узнайте chat_id: напишите боту сообщение, затем откройте `https://api.telegram.org/bot<TOKEN>/getUpdates`
3. В секции **Telegram** включите переключатель
4. Вставьте **Токен бота** и укажите **Chat ID** (можно несколько)
5. Нажмите **Тест** -- в чат придёт тестовое сообщение
6. Нажмите **Сохранить**

### MAX (VK Teams / eXpress)

1. Создайте бота в MAX и получите токен
2. В секции **MAX** включите переключатель
3. Вставьте **Токен бота** и укажите **Chat ID**
4. Нажмите **Тест** и **Сохранить**

### SNMP Trap

1. В секции **SNMP** включите переключатель
2. Выберите версию: **v2c** или **v3**
3. Для v2c: укажите **Community** (по умолчанию `public`) и адреса **Trap-приёмников**
4. Для v3: укажите **Username**, протоколы аутентификации (SHA) и шифрования (AES) с паролями
5. Нажмите **Тест** и **Сохранить**

### Webhook

1. В секции **Webhook** включите переключатель
2. Укажите один или несколько **URL** для отправки POST-запросов
3. При необходимости настройте **Заголовки** (например, `Authorization: Bearer ...`)
4. Установите **Таймаут** (по умолчанию 10 сек.)
5. Нажмите **Тест** и **Сохранить**

Формат тела webhook-запроса (JSON):

```json
{
  "event_id": "evt_...",
  "source": "guard",
  "severity": "critical",
  "title": "Обнаружен шифровальщик",
  "body": "Подробности...",
  "timestamp": "2026-04-04T12:00:00Z"
}
```

## Матрица маршрутизации

Матрица маршрутизации определяет, какие каналы получают уведомления от каждого модуля.

На странице **Уведомления** в секции **Маршрутизация** отображается таблица:

| Источник | Email | Telegram | MAX | SNMP | Webhook |
|----------|-------|----------|-----|------|---------|
| Guard | + | + | - | + | + |
| ИБП | + | + | - | + | - |
| RAID | + | + | - | + | - |
| Снапшоты | + | - | - | - | - |
| ... | | | | | |

Отметьте галочками нужные каналы для каждого источника и нажмите **Сохранить**.

## История уведомлений

Вкладка **📜 История** показывает все отправленные уведомления с возможностью фильтрации.

| Столбец | Описание |
|---------|----------|
| **Дата** | Дата и время отправки |
| **Источник** | Модуль-источник (Guard, RAID, UPS и т.д.) |
| **Severity** | Уровень: critical / warning / info |
| **Заголовок** | Краткое описание события |
| **Каналы** | Через какие каналы отправлено (Email, Telegram и т.д.) |
| **Статус** | Доставлено / Ошибка / Throttled |

Фильтры:

- По источнику (Guard, RAID, UPS, Snapshots...)
- По severity (critical, warning, info)
- По дате (с/по)
- По статусу доставки

История хранится в локальной базе данных (`/var/lib/rusnas/notify.db`). Записи старше 90 дней автоматически удаляются.

## Log Watcher

Log Watcher позволяет создавать пользовательские правила мониторинга системных логов. Если в журнале (journald) или лог-файле появляется строка, соответствующая паттерну -- отправляется уведомление.

Настройка:

1. На странице **Уведомления** откройте секцию **Log Watcher**
2. Нажмите **Добавить правило**
3. Заполните:
    - **Паттерн** -- регулярное выражение для поиска в логах
    - **Источник** -- `journald` или путь к лог-файлу
    - **Severity** -- уровень создаваемого уведомления
    - **Описание** -- текст уведомления
4. Нажмите **Сохранить**

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Тест Email не проходит | Проверьте параметры SMTP: хост, порт, логин, пароль. Убедитесь, что NAS имеет доступ к SMTP-серверу (порт не заблокирован firewall) |
| Тест Telegram не проходит | Убедитесь, что бот создан, токен верный. Проверьте, что вы написали боту хотя бы одно сообщение и chat_id указан правильно |
| Уведомления не приходят | Проверьте статус демона: `systemctl status rusnas-notifyd`. Проверьте лог: `/var/log/rusnas/notify.log` |
| Слишком много уведомлений | Настройте throttle: уменьшите `max_per_source` или увеличьте `window_sec` в конфигурации |
| Уведомления приходят с задержкой | При превышении лимита throttle уведомления группируются в дайджест. Уменьшите `digest_delay_sec` для более быстрой доставки |
