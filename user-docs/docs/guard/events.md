# Журнал событий Guard

Журнал событий содержит подробную информацию обо всех обнаруженных угрозах и подозрительной активности.

---

## Где найти

Откройте страницу **Guard** и перейдите на вкладку **"Журнал событий"**. На вкладке отображается бейдж с количеством необработанных событий.

## Таблица событий

Каждое событие в журнале содержит:

| Столбец | Описание |
|---------|----------|
| **Дата/время** | Когда произошло событие |
| **Метод** | Какой метод обнаружения сработал: Honeypot, Энтропия, IOPS, Расширение |
| **Статус** | Текущее состояние: обнаружено, заблокировано, подтверждено |
| **Путь** | Файл или директория, вызвавшие срабатывание |
| **Пользователь** | Учётная запись, от имени которой выполнялась операция |
| **Детали** | Дополнительная информация (значение энтропии, расширение файла и т.д.) |

## Фильтрация событий

Над таблицей расположена панель фильтров для быстрого поиска:

| Фильтр | Описание |
|--------|----------|
| **Метод** | Показать события только одного метода обнаружения |
| **Статус** | Фильтр по статусу: все / обнаружено / заблокировано / подтверждено |
| **Дата с** | Начальная дата периода |
| **Дата по** | Конечная дата периода |

Выберите нужные фильтры -- таблица обновится автоматически.

## Постраничная навигация

При большом количестве событий они разбиваются на страницы по 25 записей. Используйте кнопки навигации внизу таблицы для переключения между страницами.

## Подтверждение событий

После анализа события администратор может отметить его как обработанное:

1. Нажмите на событие в таблице
2. Просмотрите детали
3. Если угроза ложная или обработана -- нажмите **"Подтвердить"**

Подтверждённые события не учитываются в счётчике на бейдже вкладки.

## Разблокировка пользователя

Если Guard заблокировал пользователя (в активном или супер-безопасном режиме):

1. Найдите событие блокировки в журнале
2. Нажмите **"Разблокировать"**
3. Введите [PIN-код Guard](overview.md)
4. Пользователь будет разблокирован, и его SMB-сессия восстановлена

!!! warning "Внимание"
    Перед разблокировкой убедитесь, что угроза устранена. Если компьютер пользователя заражён шифровальщиком, разблокировка без очистки приведёт к повторной блокировке.

## Экспорт в CSV

Для анализа событий в Excel или другой программе:

1. Настройте нужные фильтры (или оставьте без фильтров для экспорта всех событий)
2. Нажмите кнопку **"Экспорт CSV"**
3. Файл будет скачан в формате CSV (кодировка UTF-8)

Экспортированный файл содержит все столбцы таблицы и может включать до 10 000 записей.

## Очистка журнала

Для удаления всех записей из журнала:

1. Нажмите кнопку **"Очистить журнал"**
2. Введите PIN-код Guard
3. Подтвердите действие

!!! danger "Важно"
    Очистка журнала необратима. Рекомендуется сначала экспортировать данные в CSV перед очисткой.

## Типичные сценарии

### Ложное срабатывание по энтропии

Некоторые легитимные файлы имеют высокую энтропию: архивы (ZIP, RAR, 7z), зашифрованные бэкапы, мультимедиа. Если вы видите частые ложные срабатывания:

- Проверьте, какие файлы вызывают срабатывание
- При необходимости увеличьте [порог энтропии](settings.md) в настройках
- Или исключите конкретные директории из мониторинга

### Реальная атака

При обнаружении реальной атаки:

1. Guard заблокирует источник (в активном режиме)
2. Автоматический снапшот будет создан (если включён)
3. Изолируйте заражённый компьютер от сети
4. Восстановите данные из [снапшота](../snapshots/manage.md)
5. После очистки компьютера разблокируйте пользователя
