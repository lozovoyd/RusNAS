# AI-ассистент

AI-ассистент позволяет управлять RusNAS через естественный язык. Вместо поиска нужных кнопок и настроек вы можете описать задачу словами, и ассистент выполнит её или подскажет, как действовать.

---

## Где найти

Откройте страницу **AI Ассистент** в боковой панели.

## Выбор AI-провайдера

RusNAS поддерживает двух AI-провайдеров:

| Провайдер | Описание | Требования |
|-----------|----------|-----------|
| **Yandex GPT** | Yandex GPT 5 Pro -- российская нейросеть | API-ключ Yandex Cloud |
| **Anthropic Claude** | Claude от Anthropic -- продвинутая языковая модель | API-ключ Anthropic |

### Настройка провайдера

1. Откройте страницу AI-ассистента
2. В настройках выберите провайдера
3. Введите API-ключ:
    - **Yandex GPT:** ключ из Yandex Cloud Console (раздел "API-ключи")
    - **Claude:** ключ из console.anthropic.com
4. Нажмите **"Сохранить"**

!!! note "Примечание"
    API-ключ хранится локально на сервере RusNAS и используется только для запросов к выбранному AI-провайдеру. Ключ не передаётся третьим лицам.

## Интерфейс чата

После настройки провайдера доступен интерфейс чата:

1. Введите вопрос или команду в поле ввода
2. Нажмите **"Отправить"** или Enter
3. Ассистент проанализирует запрос и ответит

## Примеры запросов

### Информация о системе

- "Покажи текущее состояние RAID-массивов"
- "Сколько свободного места на дисках?"
- "Какой статус Guard?"
- "Покажи последние события безопасности"

### Управление хранилищем

- "Создай SMB-шару public на /mnt/data/public"
- "Покажи все настроенные шары"
- "Какие пользователи имеют доступ к шаре documents?"

### Диагностика

- "Проверь состояние S.M.A.R.T. для всех дисков"
- "Есть ли проблемы с сетевым подключением?"
- "Покажи загрузку CPU за последний час"

### Настройка

- "Настрой автоматические снапшоты каждые 6 часов"
- "Включи дедупликацию для /mnt/data"
- "Оптимизируй параметры Samba для файлового сервера"

## 11 встроенных инструментов

AI-ассистент имеет доступ к инструментам для работы с системой:

| Инструмент | Описание |
|-----------|----------|
| Состояние RAID | Чтение статуса массивов |
| Состояние дисков | S.M.A.R.T. информация |
| Управление шарами | Создание и настройка SMB/NFS |
| Управление снапшотами | Создание, список, удаление снапшотов |
| Состояние Guard | Статус защиты и события |
| Сетевая информация | Интерфейсы, IP-адреса, маршруты |
| Системные метрики | CPU, RAM, нагрузка |
| Состояние ИБП | Заряд, статус, время автономной работы |
| Управление пользователями | Список и создание учётных записей |
| Управление сервисами | Запуск/остановка SMB, NFS, FTP |
| Диагностика сети | Ping, DNS lookup |

## Ограничения

- Ассистент не выполняет деструктивные операции (удаление массивов, форматирование) без явного подтверждения
- Скорость ответа зависит от выбранного AI-провайдера и качества интернет-соединения
- Сложные команды могут требовать уточнения

!!! warning "Внимание"
    AI-ассистент -- вспомогательный инструмент. Для критически важных операций (удаление данных, изменение RAID) всегда проверяйте действия через основной веб-интерфейс.

!!! tip "Совет"
    Формулируйте запросы максимально конкретно. Вместо "настрой хранилище" напишите "создай SMB-шару projects на томе /mnt/data с доступом для пользователей user1 и user2".
