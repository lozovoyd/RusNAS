# Проверка безопасности (Security Self-Test)

Встроенная система автоматического тестирования безопасности. Проверяет устройство на уязвимости по методологии OWASP Top 10.

---

## Назначение

Security Self-Test выполняет 47+ автоматических проверок безопасности:

- Сканирование открытых портов и сервисов
- Проверка веб-заголовков безопасности (CSP, HSTS, X-Frame-Options)
- Аудит аутентификации и паролей по умолчанию
- Поиск SQL-инъекций и XSS-уязвимостей
- Проверка конфигурации SSH, SSL/TLS
- Аудит прав доступа к файлам и директориям
- Проверка целостности системных сервисов

## Запуск проверки

### Автоматический запуск

Проверка запускается автоматически **1-го числа каждого месяца в 04:00** через systemd timer.

### Ручной запуск

Через SSH:

```bash
sudo systemctl start rusnas-sectest.service
```

Или через Cockpit Terminal.

## Результаты

Результаты сохраняются в файл отчёта и отображаются на дашборде.

### Категории проверок

| Модуль | Что проверяет |
|--------|---------------|
| **Recon** | Открытые порты, баннеры сервисов |
| **Web** | HTTP-заголовки безопасности, CSP, CORS |
| **Auth** | Пароли по умолчанию, brute-force защита |
| **Injection** | SQL-инъекции, command injection |
| **Crypto** | TLS-конфигурация, слабые шифры |
| **Access** | Права файлов, SUID/SGID, мировые записи |
| **Config** | Настройки SSH, firewall, sudo |
| **DoS** | Лимиты ресурсов, rate limiting |
| **Logging** | Аудит логирования, ротация логов |

### Уровни критичности

| Уровень | Описание | Действие |
|---------|----------|----------|
| **Critical** | Серьёзная уязвимость, требуется немедленное исправление | Исправить до следующей проверки |
| **High** | Важная проблема, может быть эксплуатирована | Запланировать исправление |
| **Medium** | Умеренный риск | Рекомендуется исправить |
| **Low** | Минимальный риск, улучшение конфигурации | По желанию |
| **Info** | Информация, не является уязвимостью | Для сведения |

## Виджет на дашборде

На дашборде отображается карточка Security с результатами последней проверки:

- Дата последнего сканирования
- Количество найденных проблем по уровням
- Общая оценка безопасности

## Просмотр отчёта

Полный отчёт доступен:

- **На дашборде** — сводка в карточке Security
- **В файле** — `/var/log/rusnas/sectest-report.json`
- **В логах** — `journalctl -u rusnas-sectest`

!!! tip "Рекомендация"
    Просматривайте результаты Security Self-Test ежемесячно. Критические и высокие уязвимости должны быть устранены в первую очередь.

!!! info "Инструменты"
    Security Self-Test использует: nmap (сканирование портов), nuclei (шаблоны уязвимостей), ffuf (перебор путей), sqlmap (SQL-инъекции), testssl.sh (проверка TLS).
