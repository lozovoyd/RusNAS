# Устранение неполадок

Справочник типичных проблем и способов их решения.

---

## Веб-интерфейс

| Проблема | Причина | Решение |
|----------|---------|---------|
| Страница не открывается | Неверный адрес или сервер недоступен | Проверьте IP-адрес и порт 9090. Убедитесь, что сервер включён и в одной сети |
| Ошибка сертификата при входе | Самоподписанный сертификат | Нажмите "Дополнительно" и "Перейти на сайт". Это безопасно для внутренней сети |
| Страница загружается без данных | Административный доступ не включён | Включите [административный доступ](getting-started/admin-access.md) (значок замка вверху) |
| Белая страница или ошибка JavaScript | Устаревший кеш браузера | Очистите кеш: Ctrl+Shift+R (Windows/Linux) или Cmd+Shift+R (macOS) |
| Интерфейс на английском | Русский язык не в приоритете браузера | Добавьте русский язык первым в настройках языка браузера |
| Таймаут при выполнении операции | Долгая операция или потеря соединения | Подождите и обновите страницу. Проверьте сетевое подключение |

## RAID и диски

| Проблема | Причина | Решение |
|----------|---------|---------|
| Массив в состоянии "Деградирован" | Диск вышел из строя | [Замените диск](raid/replace.md) как можно скорее |
| Новый диск не отображается | Диск не обнаружен системой | Нажмите "Сканировать SCSI" на странице дисков. Проверьте физическое подключение |
| Кнопка "Создать массив" неактивна | Нет свободных дисков или нет административного доступа | Убедитесь, что есть незанятые диски и включён административный доступ |
| Перестройка массива очень медленная | Высокая нагрузка на дисковую подсистему | Нормально. Перестройка работает с пониженным приоритетом, чтобы не мешать основной работе |
| Кнопка "Смонтировать" неактивна | На массиве нет файловой системы | Массив был создан без форматирования. Обратитесь к администратору для создания ФС через SSH |
| S.M.A.R.T. показывает предупреждение | Начало деградации диска | Запланируйте [замену диска](raid/replace.md). Предупреждение не означает немедленный отказ, но указывает на необходимость замены |

## Guard

| Проблема | Причина | Решение |
|----------|---------|---------|
| Забыт PIN-код Guard | -- | Сбросьте PIN через SSH: `sudo rusnas-guard --reset-pin`. Потребуется доступ к серверу по SSH |
| Guard не запускается | Ошибка конфигурации или отсутствие директорий | Проверьте, что защищаемые директории существуют и смонтированы |
| Ложные срабатывания по энтропии | Работа с архивами, видео, зашифрованными файлами | Увеличьте [порог энтропии](guard/settings.md) в настройках Guard (до 7.8-7.9) |
| Пользователь заблокирован | Guard обнаружил подозрительную активность | Проверьте событие в [журнале](guard/events.md). Если ложное срабатывание -- разблокируйте (потребуется PIN) |
| Guard не реагирует на изменения файлов | Детектор остановлен | Проверьте, что детектор запущен (кнопка "Запустить" на странице Guard) |

## Общие папки и сервисы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Не могу подключиться к SMB-шаре | Сервис SMB не запущен | Проверьте индикатор SMB на странице [Хранилище](storage/shares.md). Запустите сервис если остановлен |
| Ошибка "Отказано в доступе" при подключении | Неверные учётные данные или нет прав | Проверьте логин/пароль. Убедитесь, что пользователь имеет доступ к шаре |
| Шара пустая, хотя файлы есть | Подключение к неправильной шаре | Проверьте путь шары в настройках. Убедитесь, что том смонтирован |
| FTP не подключается | Сервис FTP не запущен или порт заблокирован | Запустите FTP на вкладке [Сервисы](storage/services.md). Проверьте порт 21 через [диагностику](network/diagnostics.md) |
| WebDAV: ошибка авторизации | Пароль WebDAV отличается от системного | WebDAV использует собственные учётные записи. Создайте пользователя в [настройках WebDAV](storage/services.md) |

## ИБП

| Проблема | Причина | Решение |
|----------|---------|---------|
| ИБП не обнаруживается | USB-кабель не подключён или ИБП не поддерживается | Проверьте USB-кабель. Нажмите "Сканировать". Проверьте совместимость модели ИБП с NUT |
| Статус "COMMLOST" (нет связи) | Потеря связи с ИБП | Проверьте USB-кабель. Перезапустите службу NUT на странице [ИБП](ups/setup.md) |
| Автоотключение не сработало | Неправильно настроены пороги | Проверьте [настройки автоотключения](ups/monitoring.md): порог заряда и времени |

## Снапшоты

| Проблема | Причина | Решение |
|----------|---------|---------|
| Снапшот не создаётся | Нет смонтированных Btrfs-томов | Убедитесь, что RAID-массив с Btrfs смонтирован |
| Нет автоматических снапшотов | Расписание не настроено или отключено | Проверьте [расписание снапшотов](snapshots/schedule.md) и убедитесь, что оно включено |
| Нет места для снапшотов | Том заполнен | Удалите старые снапшоты или увеличьте том. Проверьте использование в [анализаторе](analyzer/index.md) |

## Контейнерные приложения

| Проблема | Причина | Решение |
|----------|---------|---------|
| Установка зависает | Нет интернета или медленное соединение | Проверьте доступ к интернету. Скачивание образов может занять до 10 минут |
| Приложение не открывается после установки | Контейнер ещё запускается или ошибка | Подождите 1-2 минуты. Проверьте логи приложения. Попробуйте перезапустить |
| Ошибка "Порт занят" | Другое приложение использует порт | Измените порт при установке или остановите конфликтующее приложение |
| Недостаточно памяти для установки | Мало свободной RAM | Остановите неиспользуемые приложения. Добавьте оперативную память |

## Производительность

| Проблема | Причина | Решение |
|----------|---------|---------|
| Медленное копирование по SMB | Неоптимальные сетевые настройки | Используйте [Performance Tuner](performance/index.md) для оптимизации параметров Samba и сети |
| Высокая нагрузка CPU | Множество активных сервисов или дедупликация | Проверьте дашборд. Дедупликация работает с низким приоритетом и завершится сама |
| Сервер медленно реагирует | Мало свободной RAM, активная подкачка | Остановите неиспользуемые контейнерные приложения. Рассмотрите увеличение RAM |

## Общие рекомендации

1. **Перед обращением в поддержку** -- запишите точное сообщение об ошибке и шаги для воспроизведения
2. **Проверяйте дашборд** -- он показывает общее состояние системы и помогает быстро найти проблему
3. **Используйте AI-ассистент** -- опишите проблему в чате, ассистент может помочь с диагностикой
4. **Регулярно обновляйте** -- обновления RusNAS содержат исправления ошибок и улучшения
5. **Создавайте снапшоты** перед любыми изменениями конфигурации
