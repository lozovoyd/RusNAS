# Контейнерные приложения

RusNAS включает каталог готовых приложений, которые устанавливаются в один клик. Приложения работают в изолированных контейнерах и доступны через веб-браузер.

---

## Где найти

Откройте страницу **Приложения** в боковой панели (если доступна) или перейдите из дашборда.

## Каталог приложений

RusNAS предлагает 10 готовых приложений:

| Приложение | Категория | Описание |
|-----------|-----------|----------|
| **Nextcloud** | Файлы | Облачное хранилище, совместная работа с документами |
| **Immich** | Фото | Управление фотографиями (аналог Google Photos) |
| **Jellyfin** | Медиа | Медиасервер для видео и музыки |
| **Vaultwarden** | Безопасность | Менеджер паролей (совместим с Bitwarden) |
| **Home Assistant** | Умный дом | Управление устройствами умного дома |
| **Pi-hole** | Сеть | Блокировщик рекламы на уровне DNS |
| **WireGuard** | VPN | Быстрый и безопасный VPN-сервер |
| **Mailcow** | Почта | Почтовый сервер с веб-интерфейсом |
| **OnlyOffice** | Документы | Онлайн-редактор документов (Word, Excel, PowerPoint) |
| **Rocket.Chat** | Коммуникации | Корпоративный мессенджер |

## Установка приложения

### 1. Выбор приложения

1. В каталоге найдите нужное приложение
2. Нажмите **"Установить"**
3. Откроется модальное окно установки

### 2. Предполётная проверка (Preflight)

Система автоматически проверяет ресурсы:

| Проверка | Описание | Индикатор |
|---------|----------|-----------|
| **Оперативная память** | Достаточно ли RAM для приложения | Зелёный/Жёлтый/Красный |
| **Дисковое пространство** | Достаточно ли места на диске | Зелёный/Жёлтый/Красный |
| **Порты** | Свободны ли необходимые сетевые порты | Зелёный/Красный |

Если ресурсов недостаточно:

- Жёлтый -- установка возможна, но производительность может быть снижена
- Красный -- установка заблокирована до освобождения ресурсов

### 3. Параметры установки

В зависимости от приложения могут быть доступны настройки:

| Параметр | Описание |
|----------|----------|
| **Порт** | HTTP-порт для доступа (предлагается автоматически) |
| **Директория данных** | Где хранить данные приложения |
| **Пароль администратора** | Начальный пароль для входа в приложение |

4. Нажмите **"Установить"**

### 4. Процесс установки

Система выполнит:

1. Скачивание образов контейнеров
2. Настройка сети и хранилища
3. Запуск контейнеров
4. Настройка обратного прокси (nginx)
5. Проверка работоспособности

Прогресс отображается в модальном окне. Установка может занять от 1 до 10 минут в зависимости от размера приложения и скорости интернета.

!!! note "Примечание"
    Во время установки все образы скачиваются из интернета. Убедитесь, что сервер имеет доступ к сети.

## Доступ к установленному приложению

После установки приложение доступно по адресу:

```
http://<IP-адрес-RusNAS>/<путь-приложения>/
```

Например: `http://192.168.1.100/nextcloud/`

Точный URL показывается на карточке установленного приложения.

## Управление приложениями

Для каждого установленного приложения доступны:

| Действие | Описание |
|---------|----------|
| **Открыть** | Перейти к веб-интерфейсу приложения в новой вкладке |
| **Запустить** | Запустить остановленное приложение |
| **Остановить** | Остановить приложение (освободить ресурсы) |
| **Перезапустить** | Перезапустить при проблемах |
| **Логи** | Просмотр журнала работы для диагностики |
| **Удалить** | Полное удаление приложения и его данных |

## Просмотр логов

1. Нажмите **"Логи"** на карточке приложения
2. Откроется окно с журналом контейнера
3. Логи обновляются в реальном времени

Логи полезны для диагностики проблем при запуске или работе приложения.

## Удаление приложения

1. Нажмите **"Удалить"** на карточке приложения
2. Подтвердите удаление

!!! danger "Важно"
    Удаление приложения удаляет все его данные, включая конфигурацию, базы данных и загруженные файлы. Эта операция необратима. Перед удалением создайте резервную копию данных приложения.

## Мониторинг ресурсов

На странице приложений отображается общая статистика потребления ресурсов контейнерами:

- CPU (процент использования)
- RAM (объём потребляемой памяти)
- Дисковое пространство

!!! tip "Совет"
    Если сервер работает медленно, проверьте потребление ресурсов контейнерами. Остановите неиспользуемые приложения для освобождения памяти.
