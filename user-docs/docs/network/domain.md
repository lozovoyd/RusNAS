# Домен (Active Directory)

Интеграция rusNAS с доменом Active Directory. Два режима: присоединение к существующему домену или создание собственного контроллера домена.

---

## Где найти

Откройте **Сеть** → вкладка **🏢 Домен**.

## Выбор режима

При открытии вкладки выберите режим:

| Режим | Описание | Когда использовать |
|-------|----------|--------------------|
| **Участник домена** | NAS присоединяется к существующему AD | В компании уже есть Windows Server / Samba AD |
| **Контроллер домена** | NAS сам становится Samba AD DC | Нужен AD без отдельного Windows Server |

## Режим «Участник домена»

### Обзор

Показывает текущий статус подключения к домену: имя домена, DC, статус службы winbind.

### Подключение к домену

1. Перейдите на вкладку **🔗 Подключение**
2. Заполните:
   - **Домен** — FQDN домена (например, `company.local`)
   - **Имя администратора** — учётная запись с правами на присоединение (обычно `Administrator`)
   - **Пароль** — пароль администратора домена
3. Нажмите **Присоединиться к домену**
4. Дождитесь завершения (`net ads join`)

После присоединения доменные пользователи смогут аутентифицироваться на SMB-шарах.

### Пользователи домена

Вкладка **👥 Пользователи** показывает:

- Список пользователей домена (с поиском)
- Группы домена
- **Разрешённые группы** — только эти группы получат доступ к SMB-шарам

Чтобы разрешить группу: введите имя (например, `Domain Users`) и нажмите **+ Разрешить**.

### Настройки Samba

Вкладка **⚙️ Samba** показывает параметры `[global]` из `smb.conf`, связанные с доменом. Изменения применяются немедленно.

!!! info "Выход из домена"
    Для отключения от домена используйте команду через SSH: `sudo net ads leave -U Administrator`

## Режим «Контроллер домена»

NAS становится полноценным Samba AD DC, совместимым с Windows Active Directory.

### Создание домена

1. Перейдите на вкладку **⚙️ Создать домен**
2. Заполните:
   - **Имя домена (FQDN)** — например, `company.local`
   - **NetBIOS имя** — заполняется автоматически (например, `COMPANY`)
   - **Уровень домена** — Windows 2008 R2 / 2012 R2 / 2016
   - **DNS backend** — Samba internal (проще) или BIND9 DLZ (для production)
   - **Пароль администратора** — пароль для учётной записи `Administrator`
   - **RFC 2307** — включает POSIX UID/GID атрибуты (рекомендуется)
3. Нажмите **🚀 Создать домен**
4. Дождитесь завершения (может занять 1–3 минуты)

!!! warning "Необратимая операция"
    Создание домена перенастраивает Samba, DNS и Kerberos. Убедитесь, что параметры верны.

### Управление пользователями

Вкладка **👥 Пользователи**:

- Список всех доменных пользователей
- Кнопка **+ Создать пользователя** — имя входа, полное имя, пароль
- Поиск по имени
- Отключение / удаление пользователей

### Управление группами

Вкладка **🏢 Группы**:

- Список доменных групп
- Кнопка **+ Создать группу**
- Добавление / удаление участников

### Репликация

Вкладка **🔗 Репликация** — для среды с несколькими контроллерами домена:

- Список DC в домене
- Кнопка **🔄 Синхронизировать** — принудительная репликация
- Журнал репликации

### DNS домена

Вкладка **🌐 DNS**:

- Просмотр DNS-зон домена
- Список записей (A, AAAA, CNAME, SRV, MX)
- Кнопка **+ Добавить запись** — создание новых DNS-записей

### Групповые политики (GPO)

Вкладка **📋 Политики**:

- Список групповых политик
- Кнопка **+ Создать GPO**
- Применение GPO к организационным единицам (OU)

!!! tip "Совместимость"
    Samba AD DC совместим с Windows RSAT — вы можете управлять доменом как из rusNAS, так и из консоли Windows.
