# Сеть: интерфейсы

Страница управления сетью позволяет просматривать и настраивать сетевые интерфейсы, создавать агрегированные каналы (bonding) и VLAN, а также управлять TLS-сертификатами.

---

## Где найти

Откройте страницу **Сеть** в боковой панели.

## Список интерфейсов

На главной вкладке отображаются все сетевые интерфейсы сервера:

| Столбец | Описание |
|---------|----------|
| **Имя** | Системное имя интерфейса (например, enp0s1, eth0) |
| **Тип** | Ethernet, Bonding, VLAN, Bridge |
| **IP-адрес** | Текущий IP-адрес (или "DHCP") |
| **Маска** | Маска подсети |
| **MAC** | Аппаратный адрес |
| **Скорость** | Текущая скорость подключения |
| **Состояние** | Подключён / Отключён |

## Редактирование интерфейса

1. Нажмите на интерфейс в таблице или кнопку **"Редактировать"**
2. В модальном окне доступны настройки:

### IPv4

| Параметр | Описание |
|----------|----------|
| **Режим** | DHCP (автоматическое получение) или Статический |
| **IP-адрес** | Адрес интерфейса (только для статического режима) |
| **Маска** | Маска подсети (например, 255.255.255.0 или /24) |
| **Шлюз** | IP-адрес маршрутизатора |

### IPv6

| Параметр | Описание |
|----------|----------|
| **Режим** | Автоматический (SLAAC), DHCPv6 или Статический |
| **Адрес** | IPv6-адрес (только для статического режима) |
| **Префикс** | Длина префикса (обычно /64) |

### MTU

| Параметр | Описание |
|----------|----------|
| **MTU** | Максимальный размер пакета. По умолчанию 1500. Для jumbo frames: 9000 |

3. Нажмите **"Сохранить"**

!!! danger "Важно"
    При изменении IP-адреса интерфейса, через который вы подключены к веб-интерфейсу, соединение будет потеряно. Вам нужно будет переподключиться по новому адресу. Система предупредит вас об этом перед сохранением.

## Bonding (агрегация каналов)

Bonding объединяет несколько физических интерфейсов в один логический для повышения пропускной способности или отказоустойчивости.

### Создание Bond

1. Нажмите **"+ Создать Bond"**
2. Выберите параметры:

| Поле | Описание |
|------|----------|
| **Имя** | Имя bond-интерфейса (например, bond0) |
| **Режим** | balance-rr (балансировка), active-backup (отказоустойчивость), 802.3ad (LACP) |
| **Участники** | Выберите физические интерфейсы для объединения |

3. Настройте IP-адрес для bond-интерфейса
4. Нажмите **"Создать"**

!!! note "Примечание"
    Режим 802.3ad (LACP) требует поддержки со стороны коммутатора. Режим active-backup работает с любым коммутатором.

## VLAN

VLAN позволяет создать виртуальные сети на одном физическом интерфейсе.

### Создание VLAN

1. Нажмите **"+ Создать VLAN"**
2. Заполните:

| Поле | Описание |
|------|----------|
| **Родительский интерфейс** | Физический интерфейс для VLAN |
| **VLAN ID** | Идентификатор виртуальной сети (1-4094) |

3. Настройте IP-адрес для VLAN-интерфейса
4. Нажмите **"Создать"**

## TLS-сертификаты

В разделе сертификатов можно настроить HTTPS для веб-интерфейса и сервисов:

### Let's Encrypt

1. Нажмите **"Получить сертификат Let's Encrypt"**
2. Введите доменное имя сервера
3. Выберите метод проверки (HTTP или DNS)
4. Нажмите **"Запросить"**

!!! warning "Внимание"
    Для получения сертификата Let's Encrypt сервер должен быть доступен из интернета по порту 80 (для HTTP-проверки) или иметь доступ к DNS-провайдеру (для DNS-проверки).

### DDNS (динамический DNS)

Если у вас динамический IP-адрес, настройте DDNS для поддержания актуального доменного имени:

1. В разделе **"DDNS"** нажмите **"Настроить"**
2. Выберите провайдера (No-IP, DynDNS, Cloudflare)
3. Введите учётные данные и доменное имя
4. Нажмите **"Сохранить"**
