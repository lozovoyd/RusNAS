# Сеть: DNS и маршруты

Настройка DNS-серверов, локальных записей хостов и статических маршрутов для управления сетевой маршрутизацией.

---

## DNS-серверы

### Просмотр текущих DNS

На странице **Сеть** в разделе **"DNS"** отображаются текущие DNS-серверы, используемые системой.

### Изменение DNS-серверов

1. Нажмите **"Редактировать"** в разделе DNS
2. Введите адреса DNS-серверов (по одному в строке):

| DNS-сервер | Описание |
|-----------|----------|
| 8.8.8.8 | Google Public DNS |
| 8.8.4.4 | Google Public DNS (резервный) |
| 1.1.1.1 | Cloudflare DNS |
| 77.88.8.8 | Яндекс DNS |
| 77.88.8.1 | Яндекс DNS (резервный) |

3. Нажмите **"Сохранить"**

!!! tip "Совет"
    Укажите минимум два DNS-сервера для надёжности. Если первый сервер недоступен, система автоматически использует второй.

### Локальный DNS при использовании DHCP

Если интерфейс настроен на DHCP, DNS-серверы обычно назначаются маршрутизатором автоматически. Вы можете переопределить их вручную в настройках DNS.

## Файл /etc/hosts

Файл hosts позволяет задать соответствие имён хостов и IP-адресов на локальном уровне, без обращения к DNS-серверу.

### Просмотр записей

В разделе **"Hosts"** отображается таблица текущих записей:

| Столбец | Описание |
|---------|----------|
| **IP-адрес** | Адрес хоста |
| **Имя хоста** | Одно или несколько доменных имён, разделённых пробелами |

### Добавление записи

1. Нажмите **"+ Добавить запись"**
2. Введите IP-адрес и имя хоста
3. Нажмите **"Сохранить"**

### Удаление записи

1. Нажмите **"Удалить"** рядом с записью
2. Подтвердите действие

!!! note "Примечание"
    Записи в hosts имеют приоритет над DNS. Это полезно для тестирования или для определения локальных имён серверов без настройки DNS-сервера.

## Статические маршруты

Статические маршруты определяют, через какой шлюз достигать определённые сети.

### Когда нужны статические маршруты

- Сервер подключён к нескольким подсетям
- Необходим доступ к удалённой сети через определённый маршрутизатор
- VPN-подключение требует маршрутизации определённого трафика

### Просмотр маршрутов

В разделе **"Маршруты"** отображается таблица:

| Столбец | Описание |
|---------|----------|
| **Назначение** | Целевая сеть (например, 10.0.0.0/8) |
| **Шлюз** | IP-адрес следующего маршрутизатора |
| **Интерфейс** | Через какой сетевой интерфейс отправлять пакеты |
| **Метрика** | Приоритет маршрута (меньше = выше приоритет) |

### Добавление маршрута

1. Нажмите **"+ Добавить маршрут"**
2. Заполните:

| Поле | Описание |
|------|----------|
| **Сеть назначения** | IP-адрес сети с маской (например, 10.20.0.0/16) |
| **Шлюз** | IP-адрес маршрутизатора |
| **Интерфейс** | Выберите сетевой интерфейс |

3. Нажмите **"Сохранить"**

### Сохранение маршрутов

Добавленные через интерфейс RusNAS маршруты сохраняются и восстанавливаются после перезагрузки сервера.

### Удаление маршрута

1. Нажмите **"Удалить"** рядом с маршрутом
2. Подтвердите действие

!!! warning "Внимание"
    Не удаляйте маршрут по умолчанию (0.0.0.0/0) -- без него сервер потеряет доступ к интернету и любым сетям за пределами локальной подсети.
