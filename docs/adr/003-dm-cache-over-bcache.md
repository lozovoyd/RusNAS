# ADR-003: dm-cache (LVM) вместо bcache для SSD-кеширования

## Статус
Принято (2026-03)

## Контекст
Нужен прозрачный SSD-кеш для ускорения HDD-массивов. Два основных варианта: bcache и dm-cache (через LVM lvmcache).

## Решение
dm-cache через LVM (`lvmcache`). bcache опасен с Btrfs в режиме writeback — может привести к потере данных при сбое питания из-за конфликта между CoW (Btrfs) и write-back кешированием (bcache).

## Последствия
- ✅ Безопасен с Btrfs (writethrough по умолчанию)
- ✅ Управляется через стандартный LVM toolset
- ✅ Интеграция в существующую страницу Disks (disks.html)
- ⚠️ Чуть ниже производительность чем bcache writeback (но безопасность важнее)
- ❌ Требует LVM — дополнительный слой абстракции поверх mdadm + Btrfs
