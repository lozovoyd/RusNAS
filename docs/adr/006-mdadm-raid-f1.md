# ADR-006: Поддержка RAID F1 через mdadm

## Статус
Принято (2026-03)

## Контекст
Для сценария "backup array" нужен RAID-уровень с неравномерным распределением парности: один диск получает основную нагрузку записи (parity), остальные работают преимущественно на чтение. Это позволяет использовать один быстрый/новый диск для парности, продлевая жизнь остальным.

## Решение
RAID F1 (`--layout=raid5-f1` в mdadm) — фиксированная парность. Первый диск массива всегда содержит блок парности. Реализовано в UI как отдельная опция при создании массива.

## Последствия
- ✅ Продление жизни HDD в backup-сценарии
- ✅ Совместимость со стандартным mdadm (поддержка с ядра 3.x)
- ⚠️ Первый диск изнашивается быстрее — нужен мониторинг SMART
- ❌ Нет online-миграции с/на RAID F1 (только create)
