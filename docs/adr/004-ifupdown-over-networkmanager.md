# ADR-004: ifupdown вместо NetworkManager

## Статус
Принято (2026-03)

## Контекст
Управление сетевыми интерфейсами на NAS-сервере. NetworkManager — стандарт для десктопов, ifupdown — для серверов.

## Решение
ifupdown с прямым редактированием `/etc/network/interfaces`. NetworkManager не установлен.

## Последствия
- ✅ Предсказуемое поведение на сервере — нет фонового процесса меняющего настройки
- ✅ Нет зависимости от D-Bus
- ✅ Простая конфигурация через текстовые файлы
- ✅ Поддержка bonding, VLAN через стандартные модули
- ⚠️ Cockpit network page не работает (она для NetworkManager) — реализована своя network.html
- ❌ Нет автоматического управления Wi-Fi (не нужно для NAS)
