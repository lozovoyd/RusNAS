Каналы доставки уведомлений

Настройте каналы, через которые будут отправляться уведомления. Каждый канал можно включить/выключить и протестировать отдельно.

✉️ Email (SMTP)
Инструкция по настройке
  1. Создайте почтовый ящик на Яндексе, Mail.ru или другом SMTP-сервере
  2. Включите доступ по IMAP/SMTP в настройках почтового ящика
  3. Создайте пароль приложения (не используйте основной пароль): Яндекс — id.yandex.ru/security/app-passwords, Mail.ru — account.mail.ru/user/2-step-auth/passwords
  4. Укажите SMTP-сервер (smtp.yandex.ru для Яндекса, smtp.mail.ru для Mail.ru)
  5. Порт: 587 (STARTTLS) или 465 (SSL/TLS)
  6. В поле «Получатели» добавьте email-адреса администраторов
  7. Нажмите Тест для проверки — на указанные адреса придёт тестовое письмо
✈️ Telegram
Инструкция по настройке
  1. Откройте Telegram и найдите бота @BotFather
  2. Отправьте команду /newbot и следуйте инструкциям
  3. Скопируйте полученный токен бота (формат: 7123456789:AAH...)
  4. Добавьте бота в нужный чат или группу
  5. Для получения ID чата: добавьте бота @userinfobot в чат или перешлите ему сообщение из чата
  6. Для групп ID начинается с -100 (например: -1001234567890)
  7. Вставьте токен и ID чатов, нажмите Тест — в чат придёт тестовое сообщение
💬 MAX
Инструкция по настройке
  1. Откройте MAX (ранее VK Teams) и перейдите к созданию бота
  2. Создайте нового бота через меню настроек организации
  3. Скопируйте токен бота из настроек
  4. Добавьте бота в нужный чат
  5. Узнайте ID чата через API бота: отправьте боту сообщение и проверьте вебхук
  6. Вставьте токен и ID чатов, нажмите Тест
📊 SNMP Traps
Инструкция по настройке
  1. Убедитесь, что на сервере мониторинга (Zabbix, PRTG, Nagios) настроен приём SNMP-трапов
  2. Для SNMPv2c: укажите community string (обычно public) и адрес:порт получателя
  3. Для SNMPv3: настройте пользователя, протоколы аутентификации (SHA/MD5) и шифрования (AES/DES) с паролями
  4. Стандартный порт трапов: 162
  5. Enterprise OID rusNAS: .1.3.6.1.4.1.99999
  6. Нажмите Тест для отправки тестового трапа на указанные получатели
🔗 Webhook
Инструкция по настройке
  1. Slack: Создайте Incoming Webhook в настройках приложения Slack (api.slack.com/apps), скопируйте URL
  2. Discord: В настройках канала выберите «Интеграции» → «Вебхуки» → «Новый вебхук», скопируйте URL
  3. PagerDuty: Создайте Events API v2 интеграцию, используйте URL https://events.pagerduty.com/v2/enqueue с заголовком {"Content-Type": "application/json"}
  4. Произвольный endpoint: Укажите URL, метод (POST/PUT) и необходимые заголовки авторизации
  5. Формат тела запроса: {"source", "severity", "title", "body", "timestamp", "extra"}
  6. Нажмите Тест для проверки доставки

Защита от спама (Throttling)

Защита от спама уведомлений. Если один модуль генерирует больше N событий за указанный период — вместо отдельных уведомлений отправляется одно сводное (digest). Например: «Guard: 47 событий за 5 минут (honeypot: 30, entropy: 12, iops: 5)».
